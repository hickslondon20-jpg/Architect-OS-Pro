# .env — Reconciled Against the Code (annotated, PLACEHOLDERS ONLY)

> Every variable the code actually reads, correctly named and grouped. **Real values live only
> in the gitignored env files** (`.env.local`, `python-backend/.env`) — never in this doc.
> Derived from `python-backend/core/config.py`, `api/vcso/*.ts`, and the frontend.

## Notes / gotchas

1. **Supabase names differ per side.** Frontend uses `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY`;
   the Python backend reads `SUPABASE_URL` / `SUPABASE_SERVICE_ROLE_KEY`. Both sets are needed.
2. **Env-file location.** The backend loads `.env`/`.env.local` relative to its working dir, so
   backend vars must be in `python-backend/.env` (or launch the backend from repo root).
3. **LangSmith needs two things beyond these vars** (Ep1 Task 0): the launch must load the
   `.env` into the *process environment* (pydantic reads the file for its Settings model but
   does NOT export `LANGSMITH_*` into `os.environ`), and the Anthropic/OpenAI clients must be
   wrapped/instrumented. Env placement alone traces nothing.
4. **GKE** needs a real service-account JSON key (not keyless) — only from Ep4 onward.

---

## Frontend (Vite — root `.env` / `.env.local`; only `VITE_`-prefixed are exposed)

```
VITE_SUPABASE_URL=<your_url_here>
VITE_SUPABASE_ANON_KEY=<your_key_here>
VITE_N8N_WEBHOOK_URL=<your_webhook_url_here>
```

## Backend — REQUIRED for Gate 1 (python-backend env)

```
SUPABASE_URL=<your_url_here>                 # backend name; distinct from VITE_
SUPABASE_SERVICE_ROLE_KEY=<your_key_here>    # backend name; NOT SUPABASE_SERVICE_KEY
OPENAI_API_KEY=<your_key_here>
ANTHROPIC_API_KEY=<your_key_here>
```

## Backend — ingest bridge (set for real end-to-end smokes)

```
ARCHITECTOS_INGEST_SECRET=<your_shared_secret_here>   # same value on backend + caller
ARCHITECTOS_PYTHON_BACKEND_URL=<your_backend_base_url_here>   # caller side
```

```
ARCHITECTOS_CLAUDE_SYNTHESIS_MODEL=   # default claude-sonnet-4-6

ARCHITECTOS_METADATA_EXTRACTION_ENABLED=  # default true
ARCHITECTOS_LLM_CONTEXT_WINDOW=       # default 200000

# ARCHITECTOS_RRF_K, ARCHITECTOS_EMBEDDING_BATCH_SIZE, ARCHITECTOS_CHUNK_SIZE_TOKENS,
# ARCHITECTOS_CHUNK_OVERLAP_TOKENS, ARCHITECTOS_METADATA_MAX_INPUT_CHARS

```

## Backend — GKE sandbox (needed from Ep4 onward; NOT for Ep1–Ep2)

```
ARCHITECTOS_GKE_SERVICE_ACCOUNT_KEY=<service_account_json_or_path>   # required, not keyless
ARCHITECTOS_GCP_PROJECT_ID=<your_project_id>
ARCHITECTOS_GCP_REGION=               # default us-west2
ARCHITECTOS_GKE_CLUSTER_NAME=         # default architectos-sandbox-cluster
ARCHITECTOS_SANDBOX_IMAGE=            # default provided in config.py
ARCHITECTOS_SANDBOX_IDLE_TTL_MINUTES= # default 20
```

## N8N (batch/scheduled workflows)

```
ARCHITECTOS_WEBHOOK_SECRET=<your_secret_here>
# VITE_N8N_WEBHOOK_URL is in the frontend section above
```

```
LANGSMITH_API_KEY=<your_key_here>
LANGSMITH_PROJECT="ArchitectOS-pro"
```
