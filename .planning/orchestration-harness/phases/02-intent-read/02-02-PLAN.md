# Phase 2 Plan 02 — Adaptive Triage (lean vs. full)

> Read the workstream `../../CONTEXT.md` + `../../ROADMAP.md` and this phase's `02-CONTEXT.md` first.
> Covers **INT-2**. Consumes the classifier from `02-01`. Conservative; behind `vcso_intent_read`;
> fail-open to full assembly. This is the cost lever *on top of* Phase 1. **No router/planner.**

## Deliverable
An adaptive triage that, using the `02-01` intent classification, routes clear high-confidence
`lookup`/`ambient` turns to a **lean** assembly + lower budget with a direct-answer posture, while
`strategic_synthesis`/`brainstorm` and all low-confidence turns take the full Phase 1 working-state
assembly — proven cost-leaner on simple turns with no quality regression.

## Steps

### A. Triage gate (INT-2)
1. From the classification, select the assembly profile:
   - **Lean** (only `lookup`/`ambient`, `confidence ≥ threshold`): a smaller `assemble()` budget /
     reduced component set (e.g., skip heavy component pulls the turn doesn't need) + the direct-answer
     `systemPromptAddition`. Answer directly; minimal or no tool loop expansion.
   - **Full** (everything else, and **all** low-confidence): the Phase 1 working-state assembly path,
     unchanged.
2. The lean path reuses the Phase 1 seam with a tighter budget parameter — **no separate assembly code
   path**; it is the same `assemble()` with a leaner profile. Fail-open ⇒ full.
3. **Conservative bias:** never take the lean path on doubt. Any ambiguity, missing classification, or
   `NONE` ⇒ full assembly.

### B. Guardrails
1. The lean path must not degrade a genuinely strategic turn — if the model, mid-turn, needs more (calls
   a retrieval tool), the existing tool loop still serves it (Phase 2 does not gate tools). Lean only
   trims the *starting* assembly, it does not forbid escalation within the turn.
2. Keep the intent step + chosen profile visible through MA-05 (sanitized).

### C. Cost + quality proof (co-equal)
1. On live (canary founder, `vcso_intent_read` on), run a **mixed-intent** fixed set: several clear
   lookups/ambient + several strategic/brainstorm.
2. **Cost:** lookup/ambient turns show materially lower first-call input than the Phase-1 full baseline;
   strategic turns are unchanged; the intent pre-pass's own Haiku cost is netted in and the simple-turn
   total is still lower.
3. **Quality:** lookups answered correctly and directly; strategic still cited/sequenced/on-voice; no
   regression. A strategic turn misclassified as lookup would be a quality miss — confirm the
   conservative bias prevents it (spot-check that no strategic turn took the lean path).

## Acceptance criteria
1. High-confidence `lookup`/`ambient` → lean assembly/budget + direct answer; strategic/brainstorm and
   all low-confidence → full assembly; verified per-turn from the persisted intent + token evidence.
2. Lean uses the same `assemble()` seam (leaner profile), not a new path; fail-open to full proven.
3. Live proof: simple turns leaner (net of the pre-pass cost), strategic unchanged, **no quality
   regression**, and **no strategic turn took the lean path** on the mixed set (paired trace + DB/output).
4. `python -m compileall python-backend` clean; `ROADMAP.md`/`STATE.md` updated; `02-COMPLETION.md`
   finalized with the mixed-intent evidence table. The default flip is a separate founder call (after Phase 1's).

## Out of scope
The classifier itself (`02-01`); per-intent tool/source gating and tier escalation (P3); decomposition
and delegation (P4); reflect-and-steer (P5); flipping the flag default.
