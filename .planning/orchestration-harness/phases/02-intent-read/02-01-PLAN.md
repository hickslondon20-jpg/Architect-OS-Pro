# Phase 2 Plan 01 — Intent & Depth Classifier + Response Contract

> Read the workstream `../../CONTEXT.md` + `../../ROADMAP.md` and this phase's `02-CONTEXT.md` first.
> Covers **INT-1, INT-3**. Builds on the Phase 1 `assemble()` seam + `working_state`. Worker-tier,
> bounded, fail-open, behind `vcso_intent_read` (default off). **Verify live code/schema before changing.**

## Deliverable
A cheap worker-tier **intent-and-depth pre-pass** that runs before assembly, returns a compact
classification (move-type / depth / confidence / posture) or `NONE`, is persisted per turn, and injects
a **response contract** into the Phase 1 seam via `systemPromptAddition`. No router, no planner, no
tool/source changes.

## Steps

### A. Flag + persistence (migration/config)
1. Add platform setting `vcso_intent_read` (default **off**), mirroring the Phase 1 flag shape
   (`is_enabled` + `test_user_ids`/`enabled_for_all` + a `timeout_ms`, `circuit_breaker_max_timeouts`).
2. Choose per-turn intent persistence (execution agent's call, grounded): a small `intent jsonb` on
   `vcso_chat_messages` (consistent with `citations`/`deep_mode`) **or** a `current_intent` key in
   `working_state`. Founder-scoped by inheritance.

### B. The pre-pass (INT-1)
1. A bounded classifier on Claude Haiku via `tier_worker` (MA-06), capability key `vcso_intent_read`,
   role `utility`. Input = `working_state` (compact) + the latest message only. Output = structured
   `{move_type ∈ {lookup, strategic_synthesis, brainstorm, produce, ambient}, depth ∈ {shallow, deep},
   confidence, response_posture}`.
2. **Bounded + fail-open:** capped prompt/output/timeout; **circuit breaker** (skip after N consecutive
   timeouts, cooldown); on timeout/error/low-confidence return **`NONE`** → safe default (treat as
   `strategic_synthesis`/full, no steering). The pre-pass never breaks or delays the turn materially.
3. Persist the classification per turn (observability + P3/P4 consumption). Surface it through the MA-05
   layer as a sanitized step (curated transparency — no raw reasoning).

### C. Response contract (INT-3 + steering)
1. Map each `move_type` → a concise `systemPromptAddition` posture string (lookup → answer directly +
   briefly; brainstorm → advance the thinking, don't force a conclusion; strategic → reason with
   judgment, cite, sequence; produce → note the artifact intent; ambient → acknowledge / gently steer).
2. Inject it through the Phase 1 `assemble()` seam's `systemPromptAddition` — **no new assembly path.**
   Flag off ⇒ no injection (assembly unchanged).
3. **Injection hygiene (INT-3):** anything the pre-pass surfaces from `working_state`/message is framed
   as untrusted context, never as instructions.

## Acceptance criteria
1. `vcso_intent_read` flag present (default off); per-turn intent persisted (founder-scoped).
2. The pre-pass classifies move-type + depth + confidence before retrieval, on Haiku/`tier_worker`
   (`utility`), bounded; returns compact-or-`NONE`; circuit breaker + timeout proven (force timeouts →
   skip + safe default, turn still completes).
3. The response-contract `systemPromptAddition` reaches the model via the Phase 1 seam; flag off ⇒
   assembly byte-for-byte unchanged; a failed pre-pass leaves the turn on the full/safe path.
4. Intent step renders sanitized through MA-05 (no raw chain-of-thought).
5. `python -m compileall python-backend` clean; `ROADMAP.md`/`STATE.md` updated; `02-COMPLETION.md`
   started with evidence.

## Out of scope
Adaptive triage behavior (that is `02-02-PLAN`); router/tool-gating/source selection (P3);
decomposition/delegation (P4); reflect-and-steer (P5); flipping the flag default (a later founder call).
