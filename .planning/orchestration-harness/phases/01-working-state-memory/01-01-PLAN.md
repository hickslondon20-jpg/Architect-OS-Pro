# Phase 1 Plan 01 — Working-State Memory + Bounded Assembly Seam

> Read the workstream `../../CONTEXT.md` + `../../ROADMAP.md` and this phase's `01-CONTEXT.md` first.
> Covers **CTX-1, CTX-2, CTX-3, CTX-4**. Grounded in `vcso_chat_service.py` (`_build_context`,
> `_assemble_prompt`, `compact_thread`) and `vcso_chat_threads` (JSONB `compacted_summary` /
> `deep_resume_state` precedent). **Verify live schema/code before changing.** Behind a rollout flag.

## Deliverable
A per-thread `working_state` JSONB + a discrete `assemble()` seam that the VCSO turn uses to build a
windowed, budgeted context (working state + two-source wiki components + current move), with an
`afterTurn` working-state update, compaction demoted to fallback, all behind a default-off platform
flag and proven on live for cost + quality. No router/planner/intent logic.

## Steps

### A. Schema (migration)
1. `alter table public.vcso_chat_threads add column if not exists working_state jsonb` (nullable;
   default null). Mirror the existing `compacted_summary`/`deep_resume_state` posture + RLS (founder-scoped).
2. Add platform setting `vcso_working_state_assembly` (default **off**) via `platform_ai_settings` (or
   the existing settings mechanism) so assembly path is flag-gated. Add a worker-tier setting key for
   the update pass if it makes its own model call (reuse `tier_worker` per MA-06).

### B. Working-state artifact (CTX-1, CTX-3)
1. Define the `working_state` shape (`schema_version` + `decisions[]`, `open_questions[]`,
   `findings[]` (compact, cited), `known_unknowns[]`), each entry compact + timestamped.
2. `afterTurn` update: a bounded, worker-tier pass that folds the turn's outcome into working-state
   (add/resolve open questions, record decisions/findings). Cheap; capped size; **never writes the KB**.
   Fail-open (a failed update never breaks the turn).

### C. Assembly seam (CTX-2)
1. Extract assembly into `assemble(working_state, current_move, budget) -> {messages,
   systemPromptAddition, estimated_tokens}` (Context Hub shape). Support `isolated`/`fork` context modes
   for future worker use (interface only; workers wired later).
2. Content = working state + selected wiki components (**two-source per O2:** `wiki_*` for the fixed
   seven via `WikiReadService`, `ose_knowledge_pages` Layer-2 via `DocWikiReadService`) + current move +
   a minimal recent tail. Token-budgeted; selective/incremental (grab only needed components).
3. **Fail-open:** on any assembly error, fall back to the legacy `_build_context`/`_assemble_prompt`
   path and log (quarantine-and-downgrade). The flag off ⇒ legacy path unchanged.

### D. Compaction → fallback (CTX-4)
1. Keep `compact_thread` + `compacted_summary` as a fallback for pathological length; working-state
   assembly is the default input when the flag is on. Do not delete compaction.

### E. Rollout + proof (co-equal cost + quality)
1. Ship flag **off**; enable for the test founder on live. Run a multi-turn thread.
2. **Cost:** assembled-context tokens materially smaller than the legacy baseline (paired LangSmith
   trace + `ai_usage_log` input_tokens comparison on matched turns).
3. **Quality:** a small fixed question set answers with no regression (cited, on-voice, correct);
   working-state carries decisions/open-questions across turns (a brainstorm builds, not resets).
4. Only after both pass, recommend flipping the default (founder call).

## Acceptance criteria
1. `working_state` column live + RLS correct; flag `vcso_working_state_assembly` present (default off).
2. Working-state populates + updates across turns; contains the four field families; KB never written.
3. `assemble()` seam returns a windowed budgeted context; **fail-open to legacy verified** (force an
   assembly error, confirm the turn still completes on the legacy path).
4. Compaction still available as fallback; not the default when the flag is on.
5. Live proof: smaller assembled context vs. baseline **and** no quality regression on the fixed set,
   with paired trace + DB evidence. Flag-flip is a separate founder decision (not done here).
6. `python -m compileall python-backend` clean; frontend build green if any `src` touched (none expected);
   `ROADMAP.md`/`STATE.md` updated; `01-COMPLETION.md` written.

## Out of scope
Router (P3), planner/decomposition (P4), intent read (P2), delegation changes, wiki authoring/feeder,
the annotation grain (that is `01-02-PLAN`), and flipping the assembly default (a later founder call).
