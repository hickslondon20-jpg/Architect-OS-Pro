# Phase 4 Plan 01 — Planner Core: Decompose → Delegate → Compose

> Read the workstream `../../CONTEXT.md` + `../../ROADMAP.md` and this phase's `04-CONTEXT.md` first.
> Covers **PLAN-1, PLAN-2, PLAN-3, PLAN-4**. Reuses Ep6 `sub_agent_orchestrator` +
> `agent_capabilities` + `agent_delegation_*`; composes atop Phases 1–3. Behind `vcso_planner`
> (default off), fail-open. **Verify the live orchestrator + delegation records before changing.**

## Deliverable
A distinct **planner flow** for strategic-synthesis turns: decompose → delegate to bounded workers →
compose on the synthesis tier over compact findings, with runtime-enforced budget/depth guardrails and
nested MA-05 transparency. Reuses existing workers; adds no new worker handlers.

## Steps

### A. Flag + entry gate
1. Add `vcso_planner` (default **off**, Phase-1-flag shape). Its production flip is the last in the stack.
2. Entry gate: invoke the planner **only** when P2 intent = `strategic_synthesis` (+`deep`) and above a
   confidence threshold; **conservative default when P2 is off → do not decompose** (fall back to the P3
   path). Flag off ⇒ the P3/flat path, unchanged.

### B. Decompose (PLAN-1)
1. A synthesis-tier decomposition step: from the message + `working_state`, produce an explicit,
   **budget-bounded** set of sub-questions (cap N), each bound to a source/worker (via the P3 router's
   tier selection). The plan is recorded and **revisable mid-turn** (a worker result can add/close a
   sub-question within budget).

### C. Delegate (PLAN-2)
1. Dispatch each sub-question to a bounded worker via `sub_agent_orchestrator` (existing handlers, incl.
   `sandbox_execution_agent` for compute), on the **worker tier** (Haiku, MA-06). Parallelize
   independent sub-questions where safe.
2. Each run persists to `agent_delegation_runs`/`_steps` with `parent_run_id` = the VCSO run (existing
   linkage) → independent parent/child cost attribution + MA-05 nested rendering.
3. **Depth-bounded (PLAN-4):** one controlled level of sub-delegation; no open recursion.

### D. Compose (PLAN-3)
1. Collect the workers' **compact, cited findings** (contract in `04-02`); feed them + `working_state`
   into the Phase 1 `assemble()` seam; the **synthesis-tier main call composes** the answer over those
   compact inputs — **never raw dumps, never re-crawl.** Every claim cited.

### E. Guardrails (PLAN-4) + transparency
1. **Per-turn budget runtime-enforced:** max sub-questions, max rounds, delegation depth, spend cap —
   enforced in code, not the prompt. On budget exhaustion, compose from what was gathered (bounded).
2. **Fail-open:** any planner error → the P3/flat path; the turn never breaks.
3. Plan + each worker render through the MA-05 sub-agent surface (sanitized; no raw CoT).

## Acceptance criteria
1. `vcso_planner` present (default off); flag off ⇒ behavior unchanged; non-strategic intents never
   invoke the planner.
2. A strategic turn produces an explicit budget-bounded decomposition; workers dispatch on the worker
   tier via the existing orchestrator; runs link by `parent_run_id`; depth is bounded (no recursion).
3. Compose runs on the synthesis tier over compact findings + working state (no raw dump); answer cited.
4. Budget/depth caps enforced at runtime (proof: a cap is hit → bounded compose, not runaway).
5. Fail-open proven (forced planner error → P3/flat path).
6. Plan + workers render nested via MA-05 (sanitized). `python -m compileall python-backend` clean;
   `ROADMAP.md`/`STATE.md` updated; `04-COMPLETION.md` started.

## Out of scope
The thin-slice end-to-end proof + worker return-contract hardening (that is `04-02-PLAN`);
generalization / enabling disabled strategic workers (P6); reflect-and-steer / freshness / MCP (P5);
new worker handlers; flipping the flag default.
