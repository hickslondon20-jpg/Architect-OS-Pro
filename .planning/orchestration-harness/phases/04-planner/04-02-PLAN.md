# Phase 4 Plan 02 — Worker Return Contract + Thin-Slice Proof (checkpoint)

> Read the workstream `../../CONTEXT.md` + `../../ROADMAP.md` and this phase's `04-CONTEXT.md` first.
> Covers **PLAN-2 (return contract) + PLAN-5 (thin-slice proof + stop-and-review)**. Builds on `04-01`.
> This is the workstream's **cost-routing checkpoint**. Proof runs in the batched validation pass.

## Deliverable
A hardened **worker return contract** (compact, cited, structured findings that composition depends
on), and the **thin-slice end-to-end proof** of the full lifecycle on one hard strategic question, with
the founder stop-and-review gate.

## Steps

### A. Worker return contract (PLAN-2 detail)
1. Define/confirm the worker finding shape (reuse Ep6 `agent_delegation_*` structured output): a
   **compact** result + **claim(s)** + **evidence/citations** + **provenance** + confidence — small
   enough that the composer reasons over findings, not raw material.
2. Enforce a size cap per finding; oversized worker output is summarized to the contract, never passed
   raw into compose.
3. The `sandbox_execution_agent` finding returns the **computed result** (e.g., a ratio/trend) + how it
   was derived + source refs — the quantitative CFO input.

### B. Thin-slice proof (PLAN-5) — one hard strategic question, end to end
Run in the batched validation pass with the P1–P4 flags on for the canary founder:
1. A single hard strategic question that genuinely requires: intent = strategic (P2) → decompose (P4)
   → **≥ 2 workers including a sandbox compute over a founder dataset** → compose (synthesis tier).
2. Prove: the plan + each worker render nested via MA-05; each worker ran on the **worker tier** and
   the compose on the **synthesis tier** (per `ai_usage_log`); **independent parent/child cost
   attribution** (parent Sonnet vs. child Haiku, via `parent_run_id`); a **cited, resolved** answer;
   **no quality regression** vs. a control; and — the Phase 3 carry-forward — the strategy path is now
   **cheaper** than the flat path (compose over findings, no re-crawl). Paired LangSmith + DB/output.

### C. Stop-and-review checkpoint
1. Deliver the proof (cost + quality + transparency + attribution) to the founder as the **cost-routing
   checkpoint**. **Do not broaden** (Phase 6, disabled workers) or flip the planner default until
   reviewed.

## Acceptance criteria
1. Worker findings conform to the compact/cited/structured contract; size caps enforced; the sandbox
   worker returns a computed result with derivation + citations.
2. The thin-slice question runs end to end: intent → decompose → ≥2 workers (incl. sandbox compute) →
   compose; plan + workers visible via MA-05; correct tiers per `ai_usage_log`.
3. Independent parent/child cost attribution holds (parent Sonnet, child Haiku, linked by
   `parent_run_id`); answer cited + resolved; no quality regression.
4. Strategy path is cheaper than the flat baseline on this question (closing the Phase 3 finding),
   evidenced by paired LangSmith + DB/output.
5. Founder stop-and-review delivered; no broadening or default flip before review. `04-COMPLETION.md`
   finalized with the evidence table; `ROADMAP.md`/`STATE.md`/`Pro-Suite-Progress.md` updated.

## Out of scope
The planner flow itself (`04-01`); generalization across question types + enabling disabled strategic
workers (P6); reflect-and-steer / freshness / MCP (P5); flag flips (batched pass); new worker handlers.
