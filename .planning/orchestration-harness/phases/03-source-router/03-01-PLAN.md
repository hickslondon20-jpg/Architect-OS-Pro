# Phase 3 Plan 01 — Router Core: Tier Escalation + Recording

> Read the workstream `../../CONTEXT.md` + `../../ROADMAP.md` and this phase's `03-CONTEXT.md` first.
> Covers **ROUT-1, ROUT-3, ROUT-4**. Orchestrates existing tier readers; behind `vcso_source_router`
> (default off), fail-open to the flat tool bag. **Verify live retrieval code before changing.**

## Deliverable
A deterministic, cheapest-first **source router** that consumes the Phase 2 intent, selects a starting
tier + rule-based escalation plan over Tiers 0–3, feeds selected sources into the Phase 1 `assemble()`
seam, keeps the existing tool loop as a mid-turn escalation fallback, and records the routing decision
per turn (rendered sanitized via MA-05). No planner, no live tier, no new retrieval primitives.

## Steps

### A. Flag + tier model
1. Add `vcso_source_router` (default **off**, Phase-1-flag shape). Its production flip lands after
   Phases 1–2.
2. Define the tier model + cost order: **Tier-0** structured platform records (MRA / AE Ladder /
   sprints / Quarter Map — verify the live read path), **Tier-1** wiki components (two-source, see
   `03-02`), **Tier-2** `RetrievalService.hybrid_search`, **Tier-3** KB tools. Document a **live-tier
   hook** (Phase 5) — a no-op stub here.

### B. Rule-based escalation (ROUT-1)
1. From intent/depth (when present; conservative default when Phase 2 off) + deterministic signals,
   pick the **starting tier** and an **escalation plan**: escalate to the next tier only when the
   current tier is **insufficient** by rule — the needed component is absent, or the intent demands raw
   evidence ("what does document X say" → Tier-3), or a record lookup needs Tier-0. **No quality-score
   self-judgment** (MA-06 posture).
2. Stop at the cheapest sufficient tier; feed the selected sources into `assemble()` (P1 seam) via the
   component/context inputs (not a raw dump).
3. **Fail-open + fallback:** on router error → today's flat tool bag; and even on the router path, the
   **existing tool loop stays available** so the model can escalate mid-turn if the router under-fetched.
   Flag off ⇒ flat tool bag, byte-for-byte unchanged.

### C. Recording + transparency (ROUT-3) + isolation (ROUT-4)
1. Record the routing decision per turn (start tier, escalations taken, sources selected) — persistence
   grounded (a `routing jsonb` on the message, or in `working_state`/MA-05 steps).
2. Render it as a **sanitized MA-05 step** (which tiers were consulted and why, not raw reasoning).
3. **Founder isolation + existing tool permissions unchanged** — the router selects among sources the
   founder already may access; no broadening of scope, no cross-founder reach.

## Acceptance criteria
1. `vcso_source_router` present (default off); flag off ⇒ retrieval behavior unchanged.
2. With the flag on: record/Tier-0 questions answer from records without escalating to raw crawl;
   strategic questions start at Tier-1 components; "what does document X say" reaches Tier-3; escalation
   is rule-based and stops at the cheapest sufficient tier — verified per turn from the recorded decision.
3. Router fails open to the flat tool bag on error (forced-error proof); the model can still escalate
   mid-turn via the tool loop (proof: a router under-fetch still lets the turn reach what it needs).
4. Routing decision recorded per turn + rendered sanitized via MA-05; founder isolation intact.
5. `python -m compileall python-backend` clean; `ROADMAP.md`/`STATE.md` updated; `03-COMPLETION.md`
   started with evidence.

## Out of scope
Tier-1 two-source composition + modular founder-context (that is `03-02-PLAN`); the planner /
decompose→delegate→compose (P4); live MCP tier + freshness (P5); flipping the flag default.
