# Phase 3 Plan 02 — Tier-1 Composition: Two-Source Wiki + Modular Founder Context

> Read the workstream `../../CONTEXT.md` + `../../ROADMAP.md` and this phase's `03-CONTEXT.md` first.
> Covers **ROUT-2, ROUT-5**. Implements the Tier-1 read the `03-01` router escalates through. Honors the
> O2 resolution and the founder-context dependencies. **No planner, no live tier, no OS-Engine authoring.**

## Deliverable
Tier-1 read that **composes pre-reasoned components** rather than re-crawling raw source: the seven
fixed pages from `wiki_*` (claim/evidence — superior cited component) and emergent Layer-2 from
`ose_knowledge_pages`, over a **modular, extensible** founder-context set (business + founder-operating
pages), grabbed as needed with graceful degradation. Feeds the `03-01` router / Phase 1 seam.

## Steps

### A. Two-source wiki-component composition (ROUT-2, O2)
1. **Fixed seven (Layer-1):** read from `wiki_*` via `WikiReadService` (`per_user_wiki` reads —
   `wiki_get_page`/`wiki_get_claim`/`wiki_search`/`wiki_read_digest`), yielding compact, **cited**
   components (claim + evidence), not flat text. **Do not** depend on the `wiki_*`→OSE-Layer-1
   projection.
2. **Emergent (Layer-2):** read from `ose_knowledge_pages` via `DocWikiReadService` (`per_user_document_wiki`).
3. Compose the router-selected components into the assembly as **compact components** (selective /
   incremental — grab only what the turn needs), never a whole-wiki dump. Preserve source provenance for
   citations.

### B. Modular, extensible founder-context (ROUT-5)
1. Consume founder context as a **component set keyed by page**, not the hard-coded seven — business
   pages *and* founder-operating pages (`communication-style`, `decision-log`, goals, role, constraints)
   **when present**. Selection is by relevance to the turn/intent, not a fixed list.
2. **Graceful degradation:** a missing/absent founder-operating page is **not** a failure — compose from
   what exists. (Authoring the new pages is an OS-Engine dependency; this consumes them.)

### C. Guardrails
1. Composition is read-only; **one-writer** preserved (no wiki writes).
2. Founder isolation on all reads.
3. Keep the component selection + provenance visible through the `03-01` MA-05 routing step (sanitized).

## Acceptance criteria
1. The fixed seven compose from `wiki_*` (cited claim/evidence components) and Layer-2 from
   `ose_knowledge_pages`; **no dependency on the OSE-Layer-1 projection** (verified: the composer reads
   `wiki_*` directly for the seven).
2. Founder-context consumption is modular/extensible (not the fixed 7); a deliberately-absent
   founder-operating page degrades gracefully (turn still composes).
3. Components are compact + provenance-carrying (citations resolve); no whole-wiki dump.
4. One-writer + founder isolation intact; selection rendered sanitized via MA-05.
5. Live proof (with `03-01`): a strategic turn composes a cited answer from Tier-1 components **without**
   a full Tier-3 raw re-crawl; paired trace + DB/output. `python -m compileall python-backend` clean;
   `ROADMAP.md`/`STATE.md` updated; `03-COMPLETION.md` finalized.

## Out of scope
The escalation framework itself (`03-01`); authoring the new founder-operating pages or the OSE
projection (OS-Engine dependencies); the planner (P4); live MCP + freshness (P5); flag flip.
