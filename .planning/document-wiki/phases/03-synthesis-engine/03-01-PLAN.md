## Build — `WikiPageSynthesisService.synthesize(source_ref) -> page_id`

The generic loop, given a normalized **source payload** (adapter-produced) + the existing page corpus:
1. **Page-worthiness gate** (L2-3): is this source-material page-worthy (central / recurring), or skip?
2. **Resolve identity** via `canonical_key` — find an existing page for this entity/topic → **update**,
   else **create** (dedup; no duplicate pages).
3. **Synthesize prose** `content` with the `executive_summary` primitive (L2-10; self-rating → `confidence`),
   carrying **inline citations** + `source_file_ids[]` provenance (L2-4).
4. **Flag contradictions** vs existing pages — record, **never resolve** (CONTEXT automated/flag-don't-resolve).
5. **Set fields** — `page_kind`, `page_type` (bridge), `category`, `domain`, `word_count`, dates, `status`.
6. **Honor preserved corrections** (override) on regenerate — coordinate the hook with 06.
7. **Maintain the manifest** (`source_file_ids` ↔ registry `connected_pages`) + page links (graph).
8. **Write an activity-log event**; return the page id. (Embedding population is 05; call its hook.)

Plus the **FastAPI endpoint + async job** (`synthesis_job_id`) an adapter/event triggers. Reuse Layer 1's
service-role/transaction discipline; the engine is the only writer of synthesized `content`.

## Success criteria
1. `synthesize()` creates **or** updates a prose page with provenance; dedup via `canonical_key` works (no dup pages).
2. Page-worthiness gate skips non-worthy material.
3. Contradictions are flagged, not resolved.
4. Preserved corrections survive a re-synthesis (hook present; full lifecycle in 06).
5. Manifest + page links maintained; activity-log event written; async job + endpoint work.

## Out of scope
Source adapters (03-02 document; 04 others), embeddings/search (05), correction lifecycle + health (06).
