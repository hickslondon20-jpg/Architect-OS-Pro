## Goal

The first concrete adapter: an uploaded document → the synthesis engine. Reads the document's
`full_markdown` (+ chunks for evidence) via the KB Explorer substrate, normalizes it to a source payload,
and invokes `WikiPageSynthesisService.synthesize()`.

---

## Pre-Execution Checks
1. Read `03-01-PLAN.md` (the framework contract: what a source payload is).
2. Read `../01-verify-delta/01-01-DELTA.md` §G (KB Explorer: `full_markdown`, `match_document_chunks`,
   `ose_raw_document_registry`).
3. Confirm the document-ingest event/trigger point (`/api/ingest` completion / registry status).

## Build — `DocumentSourceAdapter`

- **Trigger:** on document-ingest completion (`ose_raw_document_registry` row reaches a ready status with
  `full_markdown`). Call the engine (async job).
- **Read evidence:** `full_markdown` for the full document; `match_document_chunks` / `kb_grep`/`kb_read`
  for targeted evidence; metadata from the registry.
- **Normalize → source payload:** document id, title, extracted text, candidate entities/topics, source refs.
- **Invoke** `synthesize()` — which may create/update one *or several* pages (a document can touch many).
- **Maintain manifest:** set `source_file_ids[]` on each touched page and `connected_pages[]` on the registry row.

## Success criteria
1. Uploading/ingesting a document fires the adapter and runs the engine.
2. Emergent page(s) are created/updated with correct `page_kind`, inline citations, and `source_file_ids` provenance.
3. The manifest (page ↔ registry) is coherent after synthesis; `ManifestView` would show the links.
4. Re-ingesting the same/updated document updates (not duplicates) pages via `canonical_key`.
5. An activity-log event records the ingest→synthesis.
