## Goal

The thread adapter (modeled on theafh `wiki_wrapup` — "synthesize the current session into pages"): a
substantive Virtual CSO thread → synthesized page(s). Built and unit-tested in v1; **live triggering from
the CSO session lifecycle is connection-phase** (build-complete, activate-as-ready).

## Build — `ThreadSourceAdapter`

- **Input:** a thread payload (thread id, messages, surfaced conclusions). Provide a **callable entry point**
  (function/endpoint) that takes a thread and runs the engine — testable in isolation with a fixture thread.
- **Synthesize:** create/update page(s) of `page_kind='thread_synthesis'` (or update an existing entity/topic
  page the thread is about); set `origin_thread_id`; provenance to the thread.
- **Page-worthiness:** only substantive threads earn pages ("within reason").
- **Activation note:** the *live* invocation (a CSO session ending → flush the thread to the adapter) is
  connection-phase — analogous to Layer 1's deferred session-end flush. Build the adapter + entry point; do
  not wire the live CSO lifecycle here.

## Success criteria
1. A fixture thread payload → page(s) created/updated of `page_kind='thread_synthesis'`, `origin_thread_id` set, provenance recorded.
2. Page-worthiness gate skips trivial threads.
3. Adapter is testable in isolation; live CSO-lifecycle trigger is explicitly left to the connection phase.

## Out of scope
Live CSO session wiring (connection phase), other adapters, embeddings (05), corrections/log/health (06).
