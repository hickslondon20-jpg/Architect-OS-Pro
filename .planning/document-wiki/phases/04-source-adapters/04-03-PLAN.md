## Goal

The domain-agent-artifact adapter: documents/outputs domain agents create → their own pages ("within
reason"). Built and unit-tested in v1; **live triggering is gated on domain-agent maturity**
(build-complete, activate-as-ready).

---

## Pre-Execution Checks
1. Read `03-01-PLAN.md` (framework).
2. Inventory how domain agents produce artifacts today (`pages/ProSuite/.../domain-agents`, any artifact
   store / `kb-files`). Confirm what an "artifact" is and where it lands — this determines the trigger
   point when activated.

## Build — `AgentArtifactSourceAdapter`

- **Input:** an agent-artifact payload (artifact id/content, originating agent + task, source refs). Provide
  a **callable entry point** testable with a fixture artifact.
- **Synthesize:** create/update a page for the artifact (`page_kind` per artifact nature — e.g. `comparison`,
  `method`, `query_answer`); provenance to the artifact + any underlying sources.
- **Page-worthiness:** only substantive artifacts earn pages.
- **Activation note:** the *live* trigger (a domain agent finishing an artifact → adapter) is connection-phase
  / gated on domain-agent readiness. Build the adapter + entry point; do not wire live agent runs here.

## Success criteria
1. A fixture artifact payload → a page created/updated with provenance to the artifact.
2. Page-worthiness gate applied; `page_kind` set sensibly per artifact type.
3. Adapter is testable in isolation; live agent-run trigger explicitly left to the connection phase.

## Out of scope
Live domain-agent wiring (connection phase), other adapters, embeddings (05), corrections/log/health (06).
