# Plan 04-01 — Sprint-History Adapter (longitudinal)

**Sub-phase:** 04 — Source Adapters
**Plan:** 1 of 3
**Depends on:** 03-01 (framework)
**Status:** Ready for execution
**Decisions:** `../../CONTEXT.md` §1 (longitudinal pages; current→L1, history→L2)

---

## Goal

The sprint adapter: each sprint gets its own page, **and** the longitudinal capability-evolution pages
accrete (the "memory of the journey"). This is the source that most distinguishes Layer 2's *historical*
role from Layer 1's *current* snapshot.

---

## Pre-Execution Checks
1. Read `03-01-PLAN.md` (framework / source payload).
2. Confirm the sprint source tables (`sp_sprint_goals/initiatives/milestones`) and the AE Ladder
   progression tables (from Layer 1's 01-DELTA §F / §A) — the capability-evolution input.
3. Confirm how Layer 1 reads *current* sprint state (so Layer 2 reads the *full history* without conflict).

## Build — `SprintSourceAdapter`

- **Trigger:** sprint events (reset / new quarter / milestone completion).
- **Per-sprint page** (`page_kind='sprint_history'`): synthesize a page capturing that sprint's goals,
  initiatives, milestones, outcomes, with provenance to the `sp_*` rows.
- **Longitudinal/evolution page(s):** update the accreting "capability evolution / focus-area journey"
  page(s) — tie to AE Ladder stage progression over quarters. `canonical_key` keeps one evolving page per arc.
- **Invoke** `synthesize()` for each; flag contradictions vs prior sprint narratives (don't resolve).

## Success criteria
1. A sprint event creates that sprint's page + updates the longitudinal capability-evolution page(s).
2. `page_kind='sprint_history'`; provenance to `sp_*` rows; canonical_key keeps the evolution page singular.
3. Layer 1 still owns *current* sprint state; Layer 2 holds the *history* — no duplication.

## Out of scope
Other adapters (04-02/03), embeddings (05), corrections/log/health (06).
