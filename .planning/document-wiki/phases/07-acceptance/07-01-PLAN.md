## Goal

Prove Layer 2 works as a **self-contained capability with no CSO/router wiring**. Deliver an automated
harness + `07-01-ACCEPTANCE.md` declaring the `doc-wiki-1.0` contract stable and ready for the
connection-phase handoff. Mirrors Layer 1's 08.

---

## Pre-Execution Checks
1. Read `../02-page-contract-schema/02-01-CONTRACT.md` (guarantees → assertions).
2. Confirm a scoped temp test user + fixtures (a sample document; a sample sprint; a fixture CSO thread;
   a fixture agent artifact). Supabase write access + (if available) OpenAI quota for real embeddings.

## The acceptance walk (assert each)

1. **Document → page.** Ingest a fixture document → emergent page(s) with `page_kind`, inline citations,
   `source_file_ids` provenance; manifest coherent.
2. **Sprint → page + longitudinal.** A sprint event → its `sprint_history` page + the capability-evolution
   page updated.
3. **Thread → page** (fixture payload) → `thread_synthesis` page with `origin_thread_id`.
4. **Artifact → page** (fixture payload) → page with artifact provenance.
5. **Dedup.** Re-running a source updates (not duplicates) via `canonical_key`.
6. **Embedding + search.** Pages embedded; `docwiki_search` returns relevant pages (real embeddings if
   quota allows; else flag the single open item, as in Layer 1).
7. **Correction preserved.** A founder correction survives a re-synthesis of the page.
8. **Contradiction flagged, not resolved.** A conflicting source → a flag, both positions kept.
9. **Health + log.** Health/lint surfaces an orphan + a stale page; the activity log records the run.
10. **Layer-1 bridge.** A Layer 2 page is reachable via the `ose_page_type` bridge without duplicating Layer 1 claims.

## Guarantee assertions
- Engine is the only writer of synthesized `content`; founder corrections are preserved overrides.
- No page duplication for the same `canonical_key`.
- Contradictions are flagged, never auto-resolved.
- RLS isolates per-user pages; no cross-user leak in search.
- No CSO/router/UI wiring introduced (capability stays isolated).

## Success criteria
1. Full scenario green (all four adapters on fixtures; dedup; search; correction; contradiction; health/log; bridge).
2. Harness runs against a scoped temp test user, not production data.
3. `07-01-ACCEPTANCE.md` records results + declares `doc-wiki-1.0` stable for the connection-phase handoff
   (embedding-quality flagged open if quota down).

## Out of scope
Any CSO / retrieval-router / UI wiring — connection phase.
