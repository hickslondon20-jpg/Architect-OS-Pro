# Plan 05-01 — Page Embeddings + Semantic Search

**Sub-phase:** 05 — Embeddings + Search
**Plan:** 1 of 1
**Depends on:** 02 (embedding column), 03 (pages being written)
**Status:** Ready for execution
**Decisions:** `../../CONTEXT.md` (pgvector; deprecate Pinecone) · **Ref:** delta §A (`document_chunks` pgvector pattern)

---

## Goal

Populate page embeddings on synthesis and build the semantic page-search path — filling the
`per_user_document_wiki` read handlers stubbed in 02. Deprecate the legacy Pinecone handle.

---

## Pre-Execution Checks
1. Read `../02-page-contract-schema/02-01-CONTRACT.md` (read tool shapes) + `02-02` (the `embedding` column).
2. Read `../01-verify-delta/01-01-DELTA.md` §A/§G (`document_chunks.embedding` + `match_document_chunks` RRF) to mirror.
3. Confirm the embedding model (OpenAI `text-embedding-3-small`, `vector(1536)` — the platform standard).

## Build

- **Embed on write:** the synthesis engine (03) embeds each page's `content` (title + key body) into
  `ose_knowledge_pages.embedding` on every create/update. Refresh on re-synthesis.
- **Search RPC / path:** a `match_wiki_pages(...)` hybrid (vector + keyword, RRF) mirroring
  `match_document_chunks`, or reuse the pattern. Returns pages with provenance + `page_kind`/metadata.
- **Fill the read handlers:** implement `docwiki_get_page`, `docwiki_search` (semantic), `docwiki_list`/index
  returning `agent_result_v1` with citations (replace the 02 stubs).
- **Deprecate Pinecone:** ignore `pinecone_vector_id`; pgvector is the only page-embedding path.

## Success criteria
1. Pages get an `embedding` on synthesis; re-synthesis refreshes it.
2. `docwiki_search` returns semantically relevant pages (verify with real embeddings if quota allows; else flag).
3. Read handlers return contract-shaped results with provenance; `docwiki_list`/index works.
4. No Pinecone dependency remains in the page path.

## Out of scope
The engine/adapters (03/04), corrections/log/health (06), CSO-hook enhancement (connection phase).
