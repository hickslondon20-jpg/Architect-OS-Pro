## Specify in `02-01-CONTRACT.md`

- **The page object** (on `ose_knowledge_pages`): which fields mean what — `page_type` (Layer-1
  bridge/area), **`page_kind` (emergent type)**, `page_title`, `content` (prose markdown with inline
  citations), `category` (UI group), `source_file_ids[]` (manifest), `canonical_key` (identity/dedup),
  `domain`, `confidence`, `status`, `word_count`, the date fields (effective/observed/review), and the
  new `embedding` (02-02).
- **`page_kind` vocabulary** — the emergent type set: `client`, `competitor`, `vendor_partner`, `offer`,
  `method`, `market_trend`, `comparison`, `query_answer`, `sprint_history`, `thread_synthesis`,
  `entity`, `concept`, … (extensible config, not a DB CHECK).
- **Provenance contract** — every page traces to sources via `source_file_ids[]` **and** inline
  citations in `content` linking to source documents/chunks. No footnote markers.
- **Manifest contract** — `ose_knowledge_pages.source_file_ids[]` ↔ `ose_raw_document_registry.connected_pages[]`
  kept coherent by the engine (no DB FK array).
- **Cross-reference/links model** — page→page links (the graph); orphans are a health finding.
- **Read tool surface** — `docwiki_get_page`, `docwiki_search` (semantic), `docwiki_list`/index. Returns
  `agent_result_v1` with citations.
- **Write surface** — internal **engine synthesis write** (the only writer of synthesized content) +
  **founder correction** (override). No freeform external page writes.
- **Layer-1 bridge** — the read contract that maps Layer 2 pages into Layer 1 contexts via `ose_page_type`
  **without duplicating** Layer 1's claim store.
- **Memory hooks** — the page is the unit of recall/promotion; name the recall/promotion fields the
  memory loop will use (recall signal, promotion state) so they're reserved now.

## Success criteria
