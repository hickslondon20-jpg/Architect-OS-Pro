## Goal

Make the contract real in the database (additive, non-breaking) and register the page-wiki tools on the
orchestrator (stub handlers). Build the engine against this in 03.

---

## Pre-Execution Checks
1. Read `02-01-CONTRACT.md`. Confirm the live `ose_knowledge_pages` schema (delta §A) before altering.
2. Read `../../wiki-system/phases/02-interface-contract/02-01-PLAN.md` for the capability-registration pattern.
3. Confirm `document_chunks` pgvector style (`vector(1536)` + HNSW) to mirror.

## Build

### Migration `docs/migrations/YYYYMMDD_docwiki_schema.sql` (additive)
- `alter table public.ose_knowledge_pages add column embedding vector(1536);` + HNSW cosine index.
- A **page-links table** `ose_page_links(user_id, from_page_id, to_page_id, relation, …)` (the graph)
  — or confirm `connected_pages`/`source_file_ids` arrays suffice; pick one and document.
- A soft **`page_kind` vocabulary** (config/`wiki_schema`-style row or config module — NOT a DB CHECK, to
  keep it extensible per CONTEXT). `page_type` CHECK stays as-is (the bridge); emergent type rides `page_kind`.
- Deprecate `pinecone_vector_id` (keep the column, stop using; comment it).
- Manifest coherence: an engine-maintained contract (app-level, or a light trigger) keeping
  `source_file_ids[]` ↔ `ose_raw_document_registry.connected_pages[]` consistent.
- **Do not break** the existing UI/CSO-hook reads — all changes additive.

### Tool registration
Register `per_user_document_wiki` (tools `docwiki_get_page`, `docwiki_search`, `docwiki_list`) on the
`agent_capabilities` registry (DB row + `_fallback_capabilities`), with `not_implemented` stub handlers
in `sub_agent_orchestrator.py` (05 fills them). Mirror Layer 1's 02 exactly.

## Success criteria
1. Migration applies; `embedding` col + HNSW index exist; existing reads unaffected.
2. `page_kind` vocabulary lives in one canonical place (extensible).
3. Links/graph representation chosen + documented; manifest-coherence mechanism documented.
4. `per_user_document_wiki` registers + dispatches to stubs returning `agent_result_v1`.
5. `pinecone_vector_id` deprecated; pgvector is the page-embedding path.
6. `generate_typescript_types` re-run.

## Out of scope
Embedding *population* + search (05), engine (03), corrections/log/health (06), UI changes.
