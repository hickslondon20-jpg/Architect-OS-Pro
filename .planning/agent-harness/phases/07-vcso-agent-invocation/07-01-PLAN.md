# Phase 7 Plan — VCSO → Domain Agent Invocation (`@Agent`)

> Read build-level `CONTEXT.md` + `ROADMAP.md`, this phase's `07-RESEARCH.md` + `07-CONTEXT.md`
> first. Honors L14, wireframe §4/§10. **No new migration.** Reuse the Phase‑2 `create_task` seam.
> Depends on Phases 1–4 + 6. Small: detection + hand-off + a thread card.

## Deliverables
1. `_detect_agent_invocation(text)` + hand-off in `vcso_chat_service.py` (pre-loop).
2. An `agent_task` SSE event/route payload carrying the spawned task ref.
3. Frontend VCSO-thread **agent-task card** (reuse `DomainAgentPrimitives`).

## Backend (`vcso_chat_service.py`, pre-loop — mirror `_detect_explicit_skill_invocation`)
- `_detect_agent_invocation(text)`: parse a leading/standalone `@AgentName …`; resolve the
  `domain_agents` row (case-insensitive key / short-name / alias). If none → normal VCSO turn.
- On match: map the remainder to a workflow via the Phase‑4 matcher; if mapped → `workflow_id`; else
  → free-form (log `freeform_requests`, `workflow_id=None`).
- `create_task(user_id, agent_id=<resolved>, workflow_id=<mapped|None>, origin='vcso',
  origin_thread_id=<thread_id>, title=<from request>)`.
- Emit an **`agent_task`** SSE event (+ include in the route payload) with `{task_id, agent, workflow,
  status, origin:'vcso'}`. **Do NOT run the task** in the VCSO loop (L14) — the founder runs it in the
  Domain Agent Workspace / it advances via Kanban.
- Persist a short assistant note referencing the spawned task (so thread history reconstructs the card).

## Frontend (VCSO thread + reuse)
- Render an inline **agent-task card** from the `agent_task` event (reuse
  `components/pro-suite/domain-agents/DomainAgentPrimitives`): agent, workflow, **status chip**,
  **Open in Workspace** (→ `/pro/intelligence/domain-agents` at that task), **View artifact**
  (Review/Done). Status = the task's Kanban state (refresh-on-open + light poll; **no inline run
  stream**).
- On thread reload, reconstruct the card from history.

## Cross-surface reconciliation
- The spawned task appears in the same Kanban + Artifacts Library with `origin='vcso'`; provenance/
  lineage identical to Profile-launched tasks. Confirm all three entry points (Profile, Kanban, VCSO)
  resolve to one task object.

## Reuse (do not rebuild)
`create_task(origin='vcso')` (Phase 2); the Phase‑4 free-form matcher + `freeform_requests`;
`DomainAgentPrimitives` + `domainAgentsApi`/`tasksApi`; the VCSO detection/route seam + SSE transport.

## Acceptance criteria
1. `@FinancialAgent …` from a VCSO thread spawns a task indistinguishable in Kanban/Library from a
   Profile-launched one except `origin='vcso'`.
2. The VCSO thread shows a handle card (status + Open-in-Workspace + View-artifact) and does **not**
   morph into a workspace (L14 verified — no inline task run in the thread).
3. All three entry points resolve to one shared task object; an unmapped `@Agent` request creates a
   free-form task + `freeform_requests` row (no crash).
4. `compileall` + focused tests + `npm.cmd run build` pass; live Anthropic smoke flagged (L18).
5. Trackers + `07-COMPLETION.md` updated.

## Out of scope
Running the task inline in VCSO (L14); L16 net-new assembly; new schema; Ep7 citation UI; visual
redesign (§8); Phase 8 verification.
