## Global platform tables (all-authenticated `select`; writes gated by existing `private.is_skill_admin()` + `service_role`)

### `domain_agents` — 5 fixed disciplines
`id uuid pk`, `key text unique` (`financial|client|operational|team|stewardship`), `name`,
`discipline_statement`, `what_its_good_at`, `icon`, `color`, `capabilities jsonb` (Analyze/Create/
Plan groupings — presentation only), `thought_starters jsonb`, `is_active bool`, `created_by uuid null`,
timestamps. **Seed all five** from vision §5 (four non-Financial disciplines + capabilities as POC
placeholders, L19).

### `templates` *(internal)*
`id`, `key`, `name`, `output_contract jsonb` (markdown structure / section schema / sizing+voice),
`render_spec jsonb` (HTML render + **Ep4 sandbox export** rules, L20), `version int`, `created_by null`,
timestamps. Seed one P&L-assessment template placeholder for Phase 3.

### `workflows`
`id`, `agent_id fk domain_agents`, `key`, `name` (plain-language), `description`, `capability text`
(Analyze/Create/Plan tag), `template_id fk templates`, `prereqs jsonb` (required-resource descriptions
for the Phase-2 prereq check), `is_active bool`, `created_by null`, timestamps. Seed one Financial
row: "Produce a Monthly P&L Assessment".

### `workflow_steps` — ordered typed steps
`id`, `workflow_id fk`, `position int`, `name`, `step_type text` CHECK
`('programmatic','llm_single','llm_agent','llm_batch_agents','llm_human_input')`,
`skill_id uuid null` (→ `skill_packs.id`; LLM steps only), `system_prompt_template text`,
`tools jsonb` (curated tool-registry id subset — D1 layer A), `capability_key text null`
(→ `agent_capabilities.capability_key`; agent/batch steps — D1 layer B, carries model routing),
`output_schema jsonb`, `workspace_inputs jsonb`, `workspace_output text`, `batch_size int null`,
timestamps. Seed the P&L step chain as ordered placeholders (Phase 3 fills prompts/skills).

## Founder-owned tables (RLS `auth.uid() = user_id`, mirror `artifacts` policies + grants)

### `tasks` — one run of a Workflow (run-state lives here, fork #1)
`id`, `user_id`, `agent_id fk`, `workflow_id fk null` (null for free-form asks), `title`,
`status text` CHECK `('ready','running','blocked','review','done')` default `'ready'`,
`current_step int` default 0, `step_results jsonb` default `'{}'`, `origin text`
CHECK `('profile','kanban','vcso')`, `origin_thread_id uuid null`, timestamps. Updated_at trigger.

### `workspace_files` — owner-flexible VFS (L21)
`id`, `owner_type text` CHECK `('task','thread')`, `owner_id uuid`, `user_id`, `file_path text`
(relative, forward-slash, no traversal, ≤500 chars), `content text null` (≤1MB), `storage_path text
null` (binary), `source text` CHECK `('agent','sandbox','upload')`, `size bigint`, timestamps.
**Unique `(owner_type, owner_id, file_path)`.** RLS by `user_id`.

### `freeform_requests` — request-capture (L16)
`id`, `user_id`, `agent_id fk`, `raw_text`, `mapped bool`, `mapped_workflow_id fk null`,
`resulting_task_id fk tasks null`, `created_at`. RLS by `user_id`.
