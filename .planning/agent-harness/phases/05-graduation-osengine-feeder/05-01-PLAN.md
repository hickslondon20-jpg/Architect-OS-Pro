# Phase 5 Plan — Graduation + OS Engine Promotion Feeder

> Read build-level `CONTEXT.md` + `ROADMAP.md`, this phase's `05-RESEARCH.md` + `05-CONTEXT.md`
> first. Honors D4, L6, L17, architecture §5 (OS Engine sole writer). **No new migration.** Backend
> + thin frontend wiring. Depends on Phases 1–4.

## Deliverables
1. `ArtifactService.register_domain_artifact(...)` — workspace `artifact.html` → `artifacts` (row +
   bucket), `source_kind='domain_agent_task'` + lineage + provenance.
2. A **graduation hook** at the engine's Review transition (register once, idempotent on revision).
3. `DocWikiAgentArtifactAdapter.synthesize_from_task(...)` — new task-sourced L17 trigger.
4. `POST /api/tasks/{id}/promote` — deliberate Add-to-Second-Brain; sets `promoted_to_kb`.
5. Frontend: wire the (inert) Second Brain controls in Workspace + Library; reflect promoted state.

## 1. Graduation — `ArtifactService.register_domain_artifact`
- Inputs: `task` (id/user_id/agent_id/workflow_id + template via workflow), the terminal
  `workspace_files` row (`owner_type='task'`, path e.g. `artifact.html`), provenance (source_refs
  from `tasks.step_results`).
- Store the workspace content in the `artifacts` bucket (founder-folder path); insert an `artifacts`
  row: `source_kind='domain_agent_task'`, `filename`, `mime_type='text/html'`, `renderable=true`,
  `task_id/workflow_id/agent_id/template_id`, `provenance`, `promoted_to_kb=false`.
- **Do NOT** reuse `deliver_from_sandbox` (sandbox-sourced, hardcodes `vcso_thread`). New method,
  workspace-sourced, no GKE.
- **Idempotent:** on Review→Running→Review (revision), update the existing row for the task, don't
  duplicate.

## 2. Graduation hook (at Review)
- When the engine transitions a task to `review` (after `task_review`), call
  `register_domain_artifact`. Keep it a thin hook so the engine stays generic; register exactly once.

## 3. L17 promotion trigger — `synthesize_from_task` (new adapter entry point)
- `DocWikiAgentArtifactAdapter.synthesize_from_task(task_id | artifact_id, user_id)`:
  assemble the artifact body from `tasks.step_results` + the registered artifact content +
  provenance; build the `DocWikiSynthesisService.synthesize(...)` payload with
  `source_kind='agent_artifact'`, `forced_page_kind='agent_artifact'`, a canonical key derived from
  the task/artifact, and source metadata. Reuse the existing synthesis service (do not fork it).
- The existing `synthesize_from_run` stays for M8 delegation-run promotion; this adds the task path.

## 4. `POST /api/tasks/{id}/promote` (founder-auth, deliberate)
- Verify task ownership + that a registered artifact exists; call `synthesize_from_task`; set
  `artifacts.promoted_to_kb=true`. Return the promotion result (OS Engine synthesis may be
  stubbed/asserted). **Never auto** — only this explicit action promotes.

## 5. Frontend wiring
- Workspace completion actions (Review/Done) + Artifacts Library row: enable **Add to Second Brain**
  → `POST /api/tasks/{id}/promote`; reflect promoted state (disable/label once promoted). Download
  stays live. Provenance link already present (Phase 4). No restyle.

## Reuse (do not rebuild)
`ArtifactService` (extend, new method); `DocWikiAgentArtifactAdapter` + `DocWikiSynthesisService`
(new entry point, reuse service); `artifacts` bucket + Phase‑1 columns; `tasks.step_results`
provenance; Phase‑4 Second Brain controls.

## Acceptance criteria
1. Reaching Review registers an `artifacts` row (`domain_agent_task`) with lineage + provenance;
   Library lists it; Download works; revision updates (not duplicates) the row.
2. `POST /{id}/promote` calls `synthesize_from_task`, sets `promoted_to_kb`; nothing auto-promotes.
3. Second Brain controls wired + reflect promoted state; skills/templates still invisible.
4. `compileall` + focused tests (fakes for OS Engine synthesis) + `npm.cmd run build` pass; live
   OS Engine/Anthropic smoke flagged deferred (L18).
5. Trackers + `05-COMPLETION.md` updated.

## Out of scope
OS Engine wiki-page generation + vectorization (OS Engine's build); sandbox DOCX export (deferred
L20 enhancement); VCSO `@Agent` (P7); Ep7 citation UI; visual redesign (§8).
