# Phase 6 Plan — Virtual CSO Deep Mode (Soft Harness)

> Read build-level `CONTEXT.md` + `ROADMAP.md`, this phase's `06-RESEARCH.md` + `06-CONTEXT.md`
> first. Honors L14, L21, L11, L12, L13, C4. **Virtual CSO only.** Additive branch on the existing
> loop; OFF = byte-for-byte current behavior. Depends on Phase 1 substrate (may run parallel to P4/P5).

## Deliverables
1. Migration `022_deep_mode_todos.sql` — `agent_todos` + `vcso_chat_messages.deep_mode`.
2. Deep-mode tools registered as deep-scoped registry natives.
3. `stream_chat` deep-mode branch (cap + tools + prompt).
4. Frontend: Deep Mode toggle + Plan Panel + Workspace Panel + inline ask_user.
5. Focused tests (fakes) + completion docs.

## Migration `022`
- `agent_todos`: `id`, `thread_id`, `user_id`, `content`, `status` CHECK
  `('pending','in_progress','completed')`, `position int`, timestamps; RLS `auth.uid()=user_id`
  (mirror the founder-owned pattern); `touch_updated_at` trigger.
- `alter table vcso_chat_messages add column if not exists deep_mode boolean not null default false`.

## Deep-mode tools (native registry, surfaced only when `deep_mode=true`)
- `write_todos(todos)` / `read_todos()` — full-replacement + recitation; → `agent_todos`
  (thread-scoped). Emit `todos_updated` SSE.
- `write_file`/`read_file`/`edit_file`/`list_files` — reuse `workspace_files` with
  `owner_type='thread'`, `owner_id=thread_id`. (No new table.)
- `task(description, context_files?)` — reuse `SubAgentOrchestrator`; clean context; shares the
  thread workspace; **no recursion**, no todos in the sub-agent.
- `ask_user(question)` — emit `ask_user` SSE, persist loop state, set
  `agent_status='waiting_for_user'`, end the stream.

## `stream_chat` branch (in `vcso_chat_service.py`)
- Read the message `deep_mode` flag. If true: `max_rounds=MAX_DEEP_ROUNDS` (~50), append deep-mode
  tools to `context["tools"]`, extend the system prompt with deep-mode instructions (deterministic,
  KV-cache-friendly — todo state via tools, not prompt). Else: **unchanged** (cap 5, no extra tools).
- Loop exhaustion → force summarize-and-deliver. Curated trace only (L11). Usage tagged
  `surface='virtual_cso'` + role (L13).
- **ask_user resume:** on a new request for a thread in `waiting_for_user`, rehydrate loop state and
  inject the reply as the `ask_user` tool result; continue. Idempotent.

## Frontend (extend, don't fork)
- `Composer`: per-message **Deep Mode toggle** next to Send; send `deep_mode` via `virtualCsoApi`.
- **Plan Panel** (editable `agent_todos`, real-time via `todos_updated`) — Deep-Mode-only (C4).
- **Workspace Panel** (thread `workspace_files`) — visible whenever the thread has files (decoupled
  from the toggle); text preview, binary download.
- Inline `ask_user` prompt; `agent_status` indicator; reconstruct panels on reload from
  `deep_mode` history.

## Reuse (do not rebuild)
VCSO `stream_chat` (branch), `tool_registry` (register natives), `workspace_files` (thread),
`SubAgentOrchestrator` (`task`), the VCSO SSE transport + `AgentStepsPanel`/`Composer`/`virtualCsoApi`.

## Acceptance criteria
1. Toggle ON grants planning/workspace/sub-agent/ask-user; **OFF byte-for-byte unchanged** (assert).
2. Todos + workspace persist + survive reload; a thread resumes after reconnect; `ask_user`
   pauses/resumes via tool-result.
3. `workspace_files` shared with Domain Agents via `owner_type` (L21 proven end to end).
4. No Deep Mode surface leaks into Domain Agents (L14); editable plan panel is Deep-Mode-only (C4).
5. `022` applied; usage tagged `virtual_cso`; `compileall` + focused tests + `npm.cmd run build`
   pass; live Anthropic smoke flagged (L18).
6. Trackers + `06-COMPLETION.md` updated.

## Out of scope
Domain Agents behavior; context auto-compaction for very long deep sessions (post-MVP);
background/async runs (deferred); VCSO `@Agent` (P7); visual redesign (§8).
