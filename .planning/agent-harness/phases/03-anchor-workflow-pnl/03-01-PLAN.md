## 1. Engine amendment — generic handler mode
In `_execute_programmatic`, add `mode='handler'`: look up `output_schema.handler` in a module-level
`PROGRAMMATIC_HANDLERS: dict[str, Callable[[EngineCtx, task, workflow, step], StepResult | BlockedResult]]`.
- A handler may return `StepResult` (advance) **or** `BlockedResult` (→ Blocked, e.g. upload prompt).
- Handlers register via a decorator/registration call at import; the engine stays generic (no P&L
  imports in the engine). Unknown handler key → `HarnessEngineError`.
- Preserve existing `copy_workspace`/`join_workspace`/static behavior unchanged.

## 2. P&L handlers
- **`pnl_intake`** — probe OS Engine for recent P&Ls (KB/structured/wiki, lightweight). If none AND
  no uploaded workspace file (`source='upload'`) → return `BlockedResult` with the upload prompt from
  `workflow.prereqs`. On an available file: parse via Docling `doc_processor` (fallback
  `structured_data`) → write `pnl-source.md` (+ parsed table JSON) to workspace. Carry `source_refs`.
- **`pnl_render`** — read `assessment.md` + template `render_spec`; render markdown→HTML **in-process**
  → write `artifact.html` to workspace. No sandbox, no `ArtifactService` (Phase 5). Return the
  artifact workspace path in the StepResult so the engine emits `task_review` with the ref.

## 3. Migration `021` — fill the seeded steps (UPDATE by workflow key + position)
For workflow `produce_monthly_pnl_assessment`:
1. **Prereq / Intake** — `output_schema` = `{"mode":"handler","handler":"pnl_intake"}`,
   `workspace_output='pnl-source.md'`.
2. **Clarify Context** — `system_prompt_template` = POC question (entity/period, one-offs, comparison
   basis), `workspace_output='review-context.md'`.
3. **Analyze P&L** — `capability_key='document_analysis_agent'`, `skill_id`=(select id from
   skill_packs where name='Diagnose the Numbers'), `workspace_inputs=['pnl-source.md','review-context.md']`,
   `workspace_output='analysis.md'`, POC `system_prompt_template`.
4. **Synthesize Assessment** — `llm_single`, `system_prompt_template` (POC),
   `output_schema` = `{ "fields": {"headline":..,"findings":[],"risks":[],"questions":[]} }` (shape per
   the `record_step_output` schema builder), `workspace_inputs=['analysis.md','review-context.md']`,
   `workspace_output='assessment.md'`.
5. **Render Artifact** — `output_schema` = `{"mode":"handler","handler":"pnl_render"}`,
   `workspace_inputs=['assessment.md']`, `workspace_output='artifact.html'`.

Mark POC content in comments so the IP pass can swap it without touching engine/wiring.
