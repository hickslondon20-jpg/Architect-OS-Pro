## `harness_engine.py`
- `create_task(user_id, agent_id, workflow_id|None, origin, origin_thread_id=None) -> task` — the
  shared seam (Phase 4 Profile/Kanban + Phase 7 `@Agent` all call this). Inserts a `tasks` row
  (`status='ready'`).
- `run_task(user_id, task_id) -> generator[{event,data}]` — the driver. Loads the task + workflow +
  ordered steps, resumes from `current_step`, dispatches each step, **persists state before
  advancing**, yields SSE items. Reuses the `/api/vcso/chat` generator+`_sse` pattern.
- **Step executors** (one strategy per `step_type`):
  - `programmatic` — in-process (Docling/`structured_data`/`structured_query`); sandbox only when
    the step contract is render/export or LLM-gen code.
  - `llm_single` — Claude call, structured output via Claude tool/JSON schema, validate vs
    `output_schema`.
  - `llm_agent` — `SubAgentOrchestrator.start_run` with `capability_key` + curated `tools[]`.
  - `llm_batch_agents` — parse items from `workspace_inputs`, chunk by `batch_size`, concurrent
    `start_run` (`asyncio.gather` / `to_thread`), stream per-item progress, accumulate to
    `workspace_output`; **resume from partial output**.
  - `llm_human_input` — emit question, set `status='blocked'`, persist, stop.
- **Context passing:** read/write `workspace_files` (`owner_type='task'`); pass **paths, not
  content**.
- **Per-step tool scope (D1):** registry `tools[]` subset (layer A) + `capability_key` via
  `AgentCapabilityScopeSource` (layer B). Never flat.
- **Prereq check:** before step 1, read OS Engine (`agent_context`/`wiki_read`) for `workflow.prereqs`;
  missing → `task_blocked` + `status='blocked'`.
- **Transitions** (persist each before next step): ready→running→blocked→review→done;
  blocked→running (reply); review→running (revision); **no running→done skip**.
- **Metering (L13):** every model call → `log_ai_usage_event(surface='domain_agents', role=…,
  model=…, task_id=…)`.
- **Curated trace (L11):** summaries only; never raw chain-of-thought.
- **Cancellation:** check a cancel flag between steps/rounds; clean stop.

## Endpoints (founder-facing → `Depends(get_current_user_id)`)
- `POST /api/tasks` → `create_task` (used internally by P4/P7 too). Returns task.
- `POST /api/tasks/{id}/run` → `StreamingResponse` over `run_task` (start or resume).
- `GET /api/tasks/{id}` → reconstruct task + workspace + step_results (reload/reconnect).
- `POST /api/tasks/{id}/messages` → founder reply that unblocks a `blocked` task (writes the
  answer to workspace, flips to running, resumes).
- `POST /api/tasks/{id}/files` → upload into `workspace_files` (`source='upload'`) to satisfy a
  prereq/resource Blocked.

(Browse/list surfaces — gallery, profile, kanban list, library — are Phase 4.)

## Migration `020_ai_usage_log_task_id.sql`
`alter table public.ai_usage_log add column if not exists task_id uuid;` + index
`ai_usage_log_task_id_idx`. No RLS change (service-role writes). Update `log_ai_usage_event(...)` to
accept + persist `task_id`.

## SSE event vocabulary (freeze — Phase 4/7 contract)
`task_ready`, `task_step_start`, `task_step_complete`, `task_step_error`, `task_blocked`,
`task_batch_progress`, `task_sub_agent_start`, `task_sub_agent_complete`, `task_review`,
`task_done`, `task_error` — all carry `task_id`; `event:/data:` framing via `_sse`.

## Reuse (do not rebuild)
`SubAgentOrchestrator.start_run`; `tool_registry` (`build_registry`, scope sources,
`ToolExecutionContext`); `sandbox_service`/`sandbox_bridge` (render/export + Code Mode only);
`_sse` + `StreamingResponse` pattern; `usage_events.log_ai_usage_event`; `structured_data`/Docling
in-process; Phase-1 `tasks`/`workspace_files`.

## Acceptance criteria
1. Trivial 2-step workflow runs end to end via the engine (state advances, workspace written, SSE
   emitted, review gate honored).
2. All 5 step modes work; `llm_human_input` blocks + resumes; `llm_batch_agents` concurrent +
   resumes from partial output.
3. Per-step tool scope is registry-subset + `capability_key` (D1); a test proves it's not a flat
   list.
4. Reconnect/`GET` reconstructs state and resumes from `current_step`.
5. `020` applied; tagged `usage_events` rows carry `surface='domain_agents'` + `role` + `task_id`.
6. Main-window overhead stays small (paths, not content) — asserted.
7. `python -m compileall python-backend` + focused tests pass; `Pro-Suite-Progress.md`/`ROADMAP.md`/
   `STATE.md` updated; `02-COMPLETION.md` written. Live smoke gaps flagged honestly.
