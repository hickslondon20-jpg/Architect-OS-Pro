## Build

### Validation checks (A7) — run post-compile and in the consolidation cycle
- **Broken provenance:** claims with zero evidence rows, or evidence whose `source_id`
  no longer resolves.
- **Frontmatter:** missing/malformed required fields per the 03-02 contract.
- **Off-taxonomy tags:** tags outside the bounded taxonomy.
- **Stale / drifted sources:** `last_compiled_at` older than source change; evidence pointing
  at superseded source versions.
- **Contested pages:** claims with open `wiki_contradictions` / `contested=true`.
- **Orphans:** claims with no page, pages with no claims.

Each check returns structured findings (claim_id / page_key / reason); **none auto-fix.**
Record-both-positions, leave resolution to the human (A3) — matches "name the tension."

### Health dashboards (B8) — generated surfaces, one set per user
- `open-questions`, `contradictions`, `low-confidence`, `claim-health`, `stale-pages`.
- Expose via a `wiki_health(user_id)` service returning all five; persist the rollup counts
  into the `wiki_digest.counts` (so the digest stays the single cheap-context object).
- **`open-questions` and `contradictions` feed the Open Questions & Unresolved Tensions page**
  — gaps and tensions surface there as questions, never as fabricated answers.

### Hook into compilation
`compile_page` (04-compilation) calls the validation set on completion; failures are logged and
surfaced on the dashboards, not silently swallowed.

---

## Success criteria
1. All six A7 checks run against real data and return structured findings without auto-fixing.
2. All five B8 dashboards generate per user; counts land in `wiki_digest`.
3. Post-compile hook runs validation automatically.
4. Open-questions + contradictions populate the Open Questions page.
5. A claim with broken provenance or an off-taxonomy tag is flagged, not dropped.

## Out of scope
Auto-resolution, dreaming-driven reconciliation (sub-phase 07), compilation (sub-phase 04), chat (connection phase).
