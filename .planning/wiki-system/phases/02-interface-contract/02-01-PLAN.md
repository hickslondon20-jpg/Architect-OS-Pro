## Goal

Formalize and **freeze** the wiki interface contract — the seam this build owns and the
later connection phase consumes. Output is a versioned contract artifact plus the
`agent_capabilities` registry rows that make `per_user_wiki` and `global_ip` real,
discoverable tools (handlers stubbed; storage lands in 03+).

Freezing the contract first de-risks the entire handoff: the wiki must not be built with a
surface the connection phase cannot consume.

## Deliverables

### 1. `02-01-CONTRACT.md` (this folder) — versioned, authoritative

Transcribe CONTEXT §4 into a standalone contract doc with a version header
(`contract_version: wiki-1.0`) and, for **every** operation, specify:
- signature, arguments (+ types), return shape, error modes;
- which **actor classes** may call it (compilation service / domain agent / founder / orchestrator-read);
- the invariant it upholds (precedence, write-lock, reasoning-only, promotion-only-path).

Operations to specify (do not change names — the connection phase binds to them):
- **Read:** `wiki_get_page`, `wiki_get_claim`, `wiki_search`, `wiki_search_insight`,
  `wiki_read_digest`, `global_ip_get`.
- **Digest schema:** the `digest{…}` object (CONTEXT §4.2), field-by-field, with how each is computed.
- **Write:** `propose_insight_claim`, `set_claim_confidence`, `flag_contradiction`,
  `add_override`, `promote_insight`.
- **Data shapes:** `claim{…}`, `evidence{…}` — the single-sourced shapes the compiler writes
  and the agent reads (CONTEXT §4.1, L5).

Include a **"Hard guarantees"** section transcribing CONTEXT §4.4 verbatim as conformance
clauses every downstream sub-phase must satisfy.

### 2. Capability registry rows

Register two new capabilities (DB rows in `agent_capabilities`, plus matching entries in the
`_fallback_capabilities()` list so the system degrades gracefully):

```
capability_key: "per_user_wiki"
  label: "Per-user wiki"
  allowed_surfaces: ["virtual_cso","os_engine","domain_agent","sprint_planning"]
  allowed_tools: ["wiki_get_page","wiki_get_claim","wiki_search",
                  "wiki_search_insight","wiki_read_digest"]
  allowed_source_kinds: ["wiki_page","wiki_claim","wiki_evidence","wiki_digest"]
  output_schema: {version: "agent_result_v1"}
  can_spawn_agents: false

capability_key: "global_ip"
  label: "ArchitectOS global IP"
  allowed_surfaces: ["virtual_cso","domain_agent","system"]
  allowed_tools: ["global_ip_get"]
  allowed_source_kinds: ["global_ip_page","global_checkpoint"]   # 01-01-DELTA §A: GM family; real kind = global_checkpoint
  output_schema: {version: "agent_result_v1"}
  can_spawn_agents: false
```

Write surface tools (`propose_insight_claim`, `set_claim_confidence`, `flag_contradiction`,
`add_override`, `promote_insight`) are defined in the contract but are **mutation endpoints**,
not retrieval-capability tools — specify their home (FastAPI service methods + the write-back
path that `api/vcso/writeback.ts` already bridges) and their actor-scope guards.

### 3. Handler stubs

In `sub_agent_orchestrator.py`, add dispatch branches for `per_user_wiki` and `global_ip`
that return a structured `not_implemented` `agent_result_v1` result. This proves registration
+ dispatch end-to-end before storage exists; sub-phases 03/04/05 fill the handlers.

**Per 01-01-DELTA §B — registering rows alone will not run the tools.** Dispatch is currently an
explicit if/elif over `document_analysis_agent` / `structured_data_agent` / `kb_explorer_agent`,
and `AgentContextBuilder` only knows `document_ids` / `chunk_ids` / `dataset_ids` / `structured_query`.
So the contract must **name the in-place orchestrator seams** the build will fill, so 03/04/05
implement against a known surface:
- `start_run()` branches `_handle_per_user_wiki()` / `_handle_global_ip()` (or replace if/elif with a handler registry, keeping the existing service);
- `AgentContextBuilder` safe-scope + loaders for page keys, claim IDs, global-IP selectors, checkpoint selectors;
- `StructuredQueryService.APPROVED_SURFACES` expansion (currently `founder_dataset_rows`, `founder_dataset_rows_v`) **or** a separate deterministic GM-checkpoint / global-IP read service.
