## Build

### The cycle `run_consolidation(user_id)` — assess → fix → verify (A5)
**Assess:** run the 06-validation-health checks + scan the insight layer for overlap/contradiction/staleness.
**Fix (write-scoped — insight layer + Open Questions ONLY):**
- **Deduplicate** overlapping insight claims (merge; log; keep evidence union).
- **Reconcile** insight-layer claims against the latest compiled base (flag where an insight is
  now contradicted or already absorbed by a compiled claim → mark superseded/retire candidate).
- **Flag contradictions** (write `wiki_contradictions`; never resolve).
- **Retire stale candidates** (status→retired with action-log entry; reversible).
- **Surface gaps as Open Questions** — append questions to the Open Questions page, never answers.
**Verify:** re-run checks until clean; write a consolidation summary + per-claim change list
to `wiki_action_log` (action='consolidate').

### Promotion candidates (B3) — surface, do not promote
Where an insight meets the machine gates (`recall_score` + `recall_count` + `query_diversity`),
set `wiki_insight_records.trust_state='promotion_candidate'`. This **surfaces** it for the
founder-confirmation surface (05-write-back `promote_insight`). Dreaming **never** calls `promote_insight`.

### Hard guardrails (assert in code + tests)
- No write path in this cycle can touch `class='compiled'`.
- No path sets `trust_state='trusted'`.
- All writes land in `wiki_action_log` and are reversible (B4).

### Scheduling
Scheduled (n8n cron per D15 default) calling the FastAPI `run_consolidation` endpoint.
Unlaunched: no founder-facing entry point; runs internally so the insight layer stays coherent
from day one.

---

## Success criteria
1. Cycle dedups, reconciles, flags, retires, and surfaces gaps — touching only insight + Open Questions.
2. Compiled base is provably untouched (test: compiled claim checksums unchanged across a run).
3. No auto-promotion occurs; only `promotion_candidate` flags are set.
4. Every change is in the action-log and reversible.
5. Runs on schedule internally with no founder-facing surface.
6. Extensibility points for the founder's forthcoming maintenance material are flagged.

## Out of scope
Auto-promotion, compiled-base writes, launching it as a named feature, chat (connection phase).
