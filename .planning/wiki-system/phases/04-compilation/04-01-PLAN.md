## Goal

Build the **single compilation service** that regenerates compiled-base pages from sources,
emits claims with line-level evidence, refreshes embeddings, and updates the digest. This is
the **only** writer of `class='compiled'` claims (L2). Eager rebuild on event (D16).

> **D15 resolution happens here.** Write the synthesis logic as a callable service
> (FastAPI endpoint/module) regardless of trigger. Default architecture to bring to London
> at execution: **n8n owns "when" (event detection, the event→rebuild map, retries,
> the existing writeback bridge), FastAPI owns "what" (this service).** The single-writer
> guarantee holds either way. Do not split claim-assembly logic across two runtimes.

## Build

### `compile_page(user_id, page_key)` — the single writer
For the given page:
1. Resolve sources per `event_rebuild_targets` / the page's source map (03-02, 01-01-DELTA §F):
   Tier 0 records and/or relevant document chunks.
2. Synthesize the compiled base as **structured claims** (not prose): each claim gets
   `text`, display `confidence` (high only when multi-source — A3), and `evidence[]` with
   `source_id, source_kind, path, lines, weight`. Use the `executive_summary` primitive (A6)
   for any prose snapshot rendering, with its self-rating as a lightweight confidence signal.
3. **Replace only the compiled-base claims** for the page (managed blocks — B7). Insight-class
   and override-class claims are **never touched** (L2). This is what makes the page
   "rebuildable AND editable."
4. Recompute and write the page `embedding` and each compiled claim `embedding` (L4).
5. Update `wiki_pages.last_compiled_at`, `stale=false`, `one_line` (A4 index entry).
6. Append a `compile` row to `wiki_action_log`.
7. Trigger digest regeneration (below).

## Success criteria
1. `compile_page` regenerates only compiled-base claims; insight + override survive untouched.
2. Every compiled claim carries ≥1 evidence row with a real `source_id`; no evidence-less claims.
3. Write-lock holds: only the compilation service writes `class='compiled'`.
4. Embeddings refresh on every compile (pages + compiled claims).
5. Each event in the map rebuilds the correct page(s), eagerly.
6. Digest reflects the latest compile.
7. Read handlers return contract-shaped results with correct precedence + visible class/trust.
8. The chosen D15 trigger/host split is documented for London's sign-off at execution time.
