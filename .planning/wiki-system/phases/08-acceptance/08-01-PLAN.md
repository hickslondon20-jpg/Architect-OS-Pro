## Goal

Prove the wiki works as a **self-contained capability with no chat/router wiring** (CONTEXT §1).
Deliver an automated harness (Vitest for TS surfaces, pytest/compileall for the Python service)
that exercises the full loop and asserts the hard guarantees. This is the gate before the
build is declared done and the connection phase picks up the frozen contract.

## The end-to-end scenario (the acceptance walk)

1. **Event → compile.** Fire a diagnostic-run event for the test user → assert Diagnostic
   Synthesis compiles with ≥1 claim, each claim carrying ≥1 line-level evidence row.
2. **Read + precedence.** `wiki_get_page('diagnostic_synthesis')` returns claims with visible
   `class`/`trust`; add a founder override → assert override precedence over the compiled claim;
   `wiki_get_claim` returns full evidence.
3. **Search.** `wiki_search(query)` returns the page across compiled+insight (L4);
   `wiki_search_insight` is correctly scoped.
4. **Digest.** `wiki_read_digest()` reflects the compile (one_line, counts, qualifiers current).
5. **Write-back.** `propose_insight_claim` on an insight-accreting page lands quarantined +
   reasoning-only; assert it is NOT assertable and NOT trusted; a compiled-base page rejects it.
6. **Promotion.** `promote_insight` (founder) → trusted; `demote_insight` reverses it; both in
   the action-log.
7. **Health.** `wiki_health` surfaces a seeded contradiction + a low-confidence + an
   orphan/broken-provenance case; Open Questions page receives the gap.
8. **Consolidation.** `run_consolidation` dedups a duplicate insight, flags a contradiction,
   retires a stale candidate, surfaces a gap — and leaves the compiled base byte-identical;
   sets a `promotion_candidate` without promoting.

## Guarantee assertions (must all pass)
- Compiled base is written **only** by the compilation service (attempt an out-of-band
  `class='compiled'` insert → rejected by trigger + API).
- Founder JWT cannot read `global_ip_pages`; service-role can.
- Quarantined insight never reaches trusted without `promote_insight`.
- Every mutation has an action-log row; every promotion is reversible.
- No code path in compilation/write-back/dreaming crosses into a forbidden layer.

## Success criteria
1. The full scenario runs green in CI-style invocation.
2. All guarantee assertions pass.
3. Harness runs against an isolated test DB/branch, not production.
4. A short `08-01-ACCEPTANCE.md` (this folder) records results and explicitly states the contract
   (`wiki-1.0`) is stable and ready for connection-phase handoff.

## Out of scope
Any chat / router / stage-primer / Domain-Agent integration — that is the connection phase.
