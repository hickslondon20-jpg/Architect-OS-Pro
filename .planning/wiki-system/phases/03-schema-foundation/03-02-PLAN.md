## Goal

Create the **Wiki Schema object** (A1): a single versioned declaration of the page set, tag
taxonomy, frontmatter contract, confidence enum, and contradiction fields — read by **both**
the compilation workflows (sub-phase 04) and the validation logic (sub-phase 06). This is the
one place the page taxonomy lives, so synthesis and validation never drift.

Adopt the *concept* of `theafh`'s `SCHEMA.md`, ported to our stack — **not** their generic
page-type enum. We keep the fixed domain page set.

## Deliverable

A versioned schema object (recommended: a typed config module **plus** a `wiki_schema`
config row in Supabase so both Python compilation and the TS frontend read the same source).

```
wiki_schema_version: "wiki-1.0"

pages:
  business_context        { kind: insight_accreting,    title: "Business Context" }
  diagnostic_synthesis    { kind: compiled_base_only,   title: "Diagnostic Synthesis" }
  current_quarter_sprint  { kind: compiled_base_only,   title: "Current Quarter / Sprint" }
  growth_constraints      { kind: insight_accreting,    title: "Growth Constraints" }
  financial_context       { kind: insight_accreting,    title: "Financial Context" }
  client_market_position  { kind: insight_accreting,    title: "Client / Market Position" }
  open_questions          { kind: insight_accreting,    title: "Open Questions & Unresolved Tensions" }

confidence_enum:  [high, medium, low]      # high reserved for multi-source (A3)
claim_status_enum: [active, quarantined, trusted, contested, retired]
claim_class_enum:  [compiled, insight, override]

frontmatter_contract:                       # required fields on every page
  - page_key            (must be in pages)
  - wiki_version
  - last_compiled_at
  - tags                (subset of tag_taxonomy)
contradiction_fields:
  - contradictions: [claim_id | page_ref]
  - contested: bool

tag_taxonomy:                               # bounded; off-taxonomy tags fail validation (A7)
  domains: [financial, team, clients, offer, operations, growth, positioning]
  stages:  [1,2,3,4,5]                       # AE Ladder: Rising/Striving/Thriving/Driving/Arriving. Model all 5; calibrated content currently 1-4, stage-5 = known gap (CONTEXT §8 2026-06-30)
  tiers:   [<1M, 1-3M, 3-6M, 6M+]            # confirm tier labels against platform in 01-01-DELTA
```

The object must expose helpers both runtimes call:
- `is_compiled_base_only(page_key)`, `is_insight_accreting(page_key)`
- `valid_tag(tag)`, `valid_page_key(page_key)`, `valid_confidence(value)`
- `event_rebuild_targets(event)` — the event→page map (filled in sub-phase 04, declared here)

## Success criteria

2. All 7 pages declared with correct `kind` (L8 split).

4. Frontmatter contract enumerated; validation (sub-phase 06) and compilation (sub-phase 04) both consume it.
5. Version string matches the contract version (`wiki-1.0`).
