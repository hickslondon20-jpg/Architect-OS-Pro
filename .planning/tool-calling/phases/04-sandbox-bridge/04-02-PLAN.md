# Plan 04-02 — Integration: Code Mode in the Sandbox Path, Shared Across Surfaces

**Requirements:** BRIDGE-02, BRIDGE-06
**Depends on:** 04-01 (the exec-channel bridge core), Phase 3 (the VCSO loop reaches the sandbox via `delegate_to_sub_agent`).
**Goal:** Wire the bridge into the existing sandbox execution path so a Code Mode run gets a session-scoped tool catalog, and confirm it is shared infrastructure (Virtual CSO now, Domain Agents later) — not a VCSO-only feature.

> Read `04-RESEARCH.md`, this phase's `CONTEXT.md`, and `04-01-PLAN.md`. Phase 3 landed: the VCSO loop already reaches sandbox execution via the registry `delegate_to_sub_agent` tool, so the integration seam is small.

---

## Step 1 — Give the sandbox execution path a scoped catalog

In `sandbox_execution_service.py` (the existing bounded code-execution sub-agent), compute the **session's scoped catalog** for Code Mode from the invoking surface/capability using the Phase 2 resolver: `registry.get_tools(surface=<invoking surface>, capability="sandbox_execution_agent")`, filtered to **read/compute tools only** — exclude `execute_code` (recursion) and any KB-write tool (one-writer). This is the allowlist the 04-01 fulfiller enforces.

## Step 2 — Route `execute_code` through `run_with_bridge`

- When the sandbox agent runs code, call the 04-01 `run_with_bridge(code, fulfiller, scoped_catalog)` instead of the plain `execute_code` path **when Code Mode is enabled for the run** — but keep the plain path for cells that need no tools (the bridge is zero-overhead when unused, so routing everything through `run_with_bridge` is acceptable if it adds no cost; confirm at build-planning whether to always-bridge or gate it).
- The system prompt for the sandbox agent gains a short note that the code may call the injected typed tool stubs (the reference's `code_mode_start` affordance) — curated, not raw. Surface a curated trace step ("running code with N tools available") consistent with Phase 3's trace vocabulary.
- Credentials, session persistence, artifact production (Ep4 Phase 6/7) all stay as-is.

## Step 3 — Confirm shared-surface reach (VCSO now, Domain Agents later)

- **Virtual CSO:** the Phase 3 loop delegates to `sandbox_execution_agent`; with 04-01+Step 2, that run now supports in-code tool calls. Verify a VCSO turn can trigger a Code Mode run whose code calls a registry tool and returns a grounded result.
- **Domain Agents:** confirm the same sandbox path is reachable by the (mock-today) Domain Agents surface without VCSO-specific assumptions — the scoped catalog derives from the *invoking* surface/capability, so Domain Agents inherit it when their live wiring lands (Ep6). Do not build Domain-Agent-specific bridge code.

## Step 4 — Trace + persistence consistency (Phase 3 ↔ Phase 7)

- A Code Mode run's tool calls are **host-side** events; surface them as curated trace steps through the existing `agent_delegation_runs`/`steps` persistence (the sandbox agent already records tool steps). Keep them curated (tool name + safe summary), never raw payloads/code — consistent with Phase 3 and L11, ready for Phase 7 reconstruction.

## Step 5 — Verify end to end

- A VCSO turn → `delegate_to_sub_agent` → sandbox Code Mode run → code calls `kb_grep`/`kb_read` via the bridge → host fulfils with credentials → result (with citation envelope) flows back into the code → final answer/artifact. Trace shows curated steps.
- Out-of-catalog / `execute_code` / write attempts from code are rejected.
- A no-tool code run is unchanged.
- Pod has zero egress; no credentials in-pod.

## Success criteria (this plan)

1. The sandbox execution path computes a session-scoped, read/compute-only catalog via the Phase 2 resolver (no `execute_code`, no KB-write) and passes it to the 04-01 bridge.
2. `execute_code` runs through `run_with_bridge` with the scoped catalog available to in-code stubs; the no-tool path is unchanged.
3. A Virtual CSO turn triggers a Code Mode run that calls a registry tool via the bridge and returns a grounded, citation-carrying result.
4. The bridge is shared: the scoped catalog derives from the invoking surface/capability, so Domain Agents inherit it later with no bridge-specific code.
5. Code Mode tool calls appear as curated trace steps persisted via `agent_delegation_runs`/`steps` (Phase 7-ready); no raw payloads/code surfaced.
6. `python -m compileall python-backend` clean; end-to-end smoke run or flagged if live creds unavailable.
