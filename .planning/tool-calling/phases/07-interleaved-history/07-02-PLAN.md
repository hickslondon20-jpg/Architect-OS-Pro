# Plan 07-02 ‚Äî Typed, Ordered, Reload-Durable Panel Rendering

**Requirements:** HIST-01, HIST-02
**Depends on:** 07-01 (the enriched step shape carrying `step_type`/`source_refs`). Frontend.
**Goal:** Reconstruct the trace as **typed, ordered panels** (tool step / sub-agent / code-execution) instead of one flat "N steps" list ‚Äî identically for live and reloaded traces, curated only, in the ArchitectOS design system.

> Read `07-RESEARCH.md` and this phase's `CONTEXT.md`. Enhance the existing `AgentStepsPanel` (lean) rather than build parallel components unless a distinct one is clearly cleaner. One rendering path fed by both live and reload. No per-round message rows.

---

## Step 1 ‚Äî Render by `step_type`, ordered

- Upgrade `AgentStepsPanel` (or add light sub-components it composes) to render steps grouped/styled by `step_type`, ordered by `step_index`:
  - **tool step** ‚Äî the current compact treatment (tool name + curated input/output summary).
  - **sub-agent** ‚Äî a nested treatment for `delegate_to_sub_agent` runs (e.g. "Knowledge Base Explorer explored N documents") showing the sub-agent's curated progress, visually distinct from a plain tool call.
  - **code-execution** ‚Äî a light typed treatment for Code Mode runs (curated: "ran code, called N tools, produced <artifact>"), linked to the artifact card. **No raw code/stdout** (L11 + the "don't show the guts" vision).
- Keep it collapsed-by-default (as today) with the ordered typed panels revealed on expand.

## Step 2 ‚Äî One path for live and reload

- Ensure the live SSE trace (Phase 3 `step`/`tool_call`/`tool_result`, now carrying `step_type` from 07-01) and the reloaded `agentSteps` render through the **same** component with the **same** typed treatment ‚Äî so a thread looks identical mid-stream and after reload.
- `MessageBubble` composes: typed trace panel (ordered) ‚Üí the single final answer text ‚Üí artifact cards. (Our loop defers the answer to one block, so this order is faithful ‚Äî no per-round text interleaving.)

## Step 3 ‚Äî Curated only (L11)

- No raw payloads, code, stdout, or model reasoning on reload or live. Use the curated `input`/`output`/`summary`/`title`. `source_refs` is carried in the data (for the future citations episode) but **not rendered as citations here**.

## Step 4 ‚Äî Design system

- ArchitectOS tokens throughout (Obsidian/Brass/Parchment; Geist/Geist Mono for the mono trace labels, as `AgentStepsPanel` already uses `aos-mono`); no Inter, no default grays. Typed panels should feel like one coherent trace, not a new UI.

## Step 5 ‚Äî Verify

- A multi-tool turn with a sub-agent delegation and a Code Mode run: on reload, the panels reconstruct **typed and in order** (tool steps, the sub-agent panel, the code-execution panel), above the final answer, with artifact cards intact.
- Live streaming of the same turn renders the identical typed panels as they arrive.
- Everything is curated (no raw code/payloads); design tokens; no visual regression from the current flat panel for simple turns.

## Success criteria (this plan)

1. Reloaded traces render as typed, ordered panels (tool / sub-agent / code-execution), not one flat list; ordered by `step_index`; above the single final answer with artifact cards intact.
2. Live and reloaded traces render identically through one rendering path.
3. Curated only ‚Äî no raw chain-of-thought, code, stdout, or payloads (L11); `source_refs` carried but not rendered as citations (that's the later episode).
4. ArchitectOS design tokens; no regression for simple single-tool turns.
5. `npm run build` clean; live + reload verified or gaps flagged honestly.
