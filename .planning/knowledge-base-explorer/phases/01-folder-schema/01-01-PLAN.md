## Schema

### `kb_folders` Table

```sql
CREATE TABLE IF NOT EXISTS public.kb_folders (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  name        TEXT NOT NULL CHECK (char_length(name) > 0 AND char_length(name) <= 255),
  parent_id   UUID REFERENCES public.kb_folders(id) ON DELETE CASCADE,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

**Design notes:**
- `parent_id NULL` = root-level folder (no parent)
- `parent_id` self-referencing FK with `ON DELETE CASCADE` — deleting a parent deletes all children recursively (Postgres handles this via deferred constraint evaluation)
- `name` constraint prevents empty strings and enforces a sane max length
- No `is_global` or `visibility` column — per-user only by design

### Indexes

```sql
-- Primary lookup: all folders for a given user
CREATE INDEX IF NOT EXISTS kb_folders_user_id_idx ON public.kb_folders (user_id);

-- Folder tree traversal: children of a given parent
CREATE INDEX IF NOT EXISTS kb_folders_parent_id_idx ON public.kb_folders (parent_id);

-- Combined: user's folders by parent (most common query pattern)
CREATE INDEX IF NOT EXISTS kb_folders_user_parent_idx ON public.kb_folders (user_id, parent_id);
```

### Updated_at Trigger

```sql
CREATE OR REPLACE FUNCTION public.update_kb_folders_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER kb_folders_updated_at_trigger
  BEFORE UPDATE ON public.kb_folders
  FOR EACH ROW
  EXECUTE FUNCTION public.update_kb_folders_updated_at();
```

## Row-Level Security Policies

```sql
-- Enable RLS
ALTER TABLE public.kb_folders ENABLE ROW LEVEL SECURITY;

-- SELECT: users can only see their own folders
CREATE POLICY "kb_folders_select_own"
  ON public.kb_folders
  FOR SELECT
  USING (auth.uid() = user_id);

-- INSERT: users can only create folders for themselves
CREATE POLICY "kb_folders_insert_own"
  ON public.kb_folders
  FOR INSERT
  WITH CHECK (auth.uid() = user_id);

-- UPDATE: users can only update their own folders
CREATE POLICY "kb_folders_update_own"
  ON public.kb_folders
  FOR UPDATE
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

-- DELETE: users can only delete their own folders
CREATE POLICY "kb_folders_delete_own"
  ON public.kb_folders
  FOR DELETE
  USING (auth.uid() = user_id);
```

## Verification Steps

After applying the migration via Supabase MCP or SQL editor:

1. **Table exists:**
   ```sql
   SELECT column_name, data_type, is_nullable
   FROM information_schema.columns
   WHERE table_name = 'kb_folders'
   ORDER BY ordinal_position;
   ```
   Expected: id, user_id, name, parent_id, created_at, updated_at

2. **RLS is on:**
   ```sql
   SELECT relrowsecurity FROM pg_class WHERE relname = 'kb_folders';
   ```
   Expected: `t`

3. **Policies exist:**
   ```sql
   SELECT policyname, cmd FROM pg_policies WHERE tablename = 'kb_folders';
   ```
   Expected: 4 policies (select, insert, update, delete)

4. **Indexes exist:**
   ```sql
   SELECT indexname FROM pg_indexes WHERE tablename = 'kb_folders';
   ```
   Expected: primary key + 3 named indexes

5. **Self-referencing cascade:** Insert a parent folder, insert a child, delete the parent — child should be gone.
