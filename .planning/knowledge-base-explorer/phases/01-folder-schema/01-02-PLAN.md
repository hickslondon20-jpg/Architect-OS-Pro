## New File: `python-backend/routers/kb_folders.py`

### Pydantic Models

```python
from pydantic import BaseModel, Field
from typing import Optional
from uuid import UUID
from datetime import datetime


class FolderCreate(BaseModel):
    name: str = Field(..., min_length=1, max_length=255)
    parent_id: Optional[UUID] = None


class FolderRename(BaseModel):
    name: str = Field(..., min_length=1, max_length=255)


class FolderResponse(BaseModel):
    id: UUID
    user_id: UUID
    name: str
    parent_id: Optional[UUID]
    created_at: datetime
    updated_at: datetime

    class Config:
        from_attributes = True
```

### Router Endpoints

#### POST `/kb/folders` — Create folder

```
Request body: FolderCreate { name, parent_id? }
Auth: Bearer JWT (Supabase)
Response 201: FolderResponse
Response 400: parent_id not found or belongs to different user
Response 401: missing/invalid token
```

Logic:
1. Extract `user_id` from JWT
2. If `parent_id` provided, verify it exists AND belongs to `user_id` (prevents cross-user parent injection)
3. Insert into `kb_folders` returning the new row
4. Return `FolderResponse`

#### PATCH `/kb/folders/{folder_id}` — Rename folder

```
Request body: FolderRename { name }
Auth: Bearer JWT (Supabase)
Response 200: FolderResponse
Response 404: folder not found or not owned by user
Response 401: missing/invalid token
```

Logic:
1. Extract `user_id` from JWT
2. UPDATE where `id = folder_id AND user_id = user_id` returning updated row
3. If no row updated → 404
4. Return `FolderResponse`

#### DELETE `/kb/folders/{folder_id}` — Delete folder

```
Auth: Bearer JWT (Supabase)
Response 204: no content
Response 404: folder not found or not owned by user
Response 401: missing/invalid token
```

Logic:
1. Extract `user_id` from JWT
2. DELETE where `id = folder_id AND user_id = user_id`
3. If no row deleted → 404
4. Return 204 (cascade handles children and future doc cleanup in Phase 2)

#### GET `/kb/folders` — List folders (flat list)

```
Query params: parent_id? (UUID, optional — filter by parent)
Auth: Bearer JWT (Supabase)
Response 200: List[FolderResponse]
Response 401: missing/invalid token
```

Logic:
1. Extract `user_id` from JWT
2. SELECT where `user_id = user_id` AND `parent_id = parent_id` (or `parent_id IS NULL` for root if param omitted and explicit root requested)
3. If `parent_id` param is absent, return ALL folders for the user (flat list — UI builds the tree)
4. Return `List[FolderResponse]` ordered by `name ASC`

> **Note on tree vs. flat:** The GET endpoint returns a flat list. The frontend builds the visual tree from the flat list using parent_id references. The Python `tree` tool in Phase 4 will handle the hierarchical format for agents. Do not build a recursive tree response here.

## Auth Pattern

Use whatever `get_current_user` or JWT extraction dependency already exists in the Python backend. Do not create a new auth pattern. If no dependency exists, create one that:
1. Reads `Authorization: Bearer <token>` header
2. Validates the JWT against Supabase's JWKS or decodes it using the Supabase JWT secret from config
3. Returns `user_id` (UUID) or raises `HTTPException(401)`

   Attempt to create a folder with `parent_id` belonging to a different user's test folder.
   → Must return 400, not 201.

9. **Invalid token:**
   Send any request without `Authorization` header → 401.
