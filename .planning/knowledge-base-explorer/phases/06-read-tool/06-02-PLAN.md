## Step 2 — New Pydantic Models

Add after the `KbGlobResponse` model (currently the last KB model) and before `AgentCapabilityResponse`:

```python
class KbReadRequest(BaseModel):
    user_id: str = Field(..., min_length=1)
    document_id: str = Field(..., min_length=1)
    start_line: int | None = Field(default=None, ge=1)
    end_line: int | None = Field(default=None, ge=1)

    @model_validator(mode="after")
    def validate_line_range(self) -> "KbReadRequest":
        start = self.start_line
        end = self.end_line
        # Both must be provided together or both omitted
        if (start is None) != (end is None):
            raise ValueError(
                "Both start_line and end_line must be provided together, or both omitted."
            )
        if start is not None and end is not None:
            if end < start:
                raise ValueError("end_line must be >= start_line.")
            if end - start + 1 > 500:
                raise ValueError(
                    f"Line range too large: {end - start + 1} lines requested; maximum is 500."
                )
        return self


class KbReadResponse(BaseModel):
    document_id: str
    name: str
    total_lines: int
    start_line: int
    end_line: int
    truncated: bool
    content: str
```

## Step 3 — New Endpoint Function

Add after the `kb_glob` function and before `@app.get("/api/agent-capabilities"...)`:

```python
@app.post("/api/tools/kb-read", response_model=KbReadResponse, dependencies=[Depends(require_ingest_secret)])
def kb_read(payload: KbReadRequest) -> KbReadResponse:
    try:
        result = KbNavigationService.from_env().execute_read(
            user_id=payload.user_id,
            document_id=payload.document_id,
            start_line=payload.start_line,
            end_line=payload.end_line,
        )
    except KbNavigationError as exc:
        raise HTTPException(status_code=status.HTTP_404_NOT_FOUND, detail=str(exc)) from exc
    except VectorStoreError as exc:
        raise HTTPException(status_code=status.HTTP_502_BAD_GATEWAY, detail=str(exc)) from exc
    return KbReadResponse(**read_result_to_dict(result))
```

## Notes

- `KbNavigationError` → 404 for this endpoint (same as ls/tree, different from grep's 400). This is correct — "document not found" and "content not yet available" are both 404 resource conditions, not client input errors.
- The `@model_validator(mode="after")` runs after Pydantic sets field values. In `mode="after"`, `self` is the already-constructed model instance — access fields as `self.start_line`, not `values["start_line"]`.
- `model_validator` import: check whether main.py already imports it from pydantic. If `from pydantic import BaseModel, Field` exists, extend to `from pydantic import BaseModel, Field, model_validator`. Do not duplicate the import.
- `npm run build` is NOT required — Phase 6 is Python-only.
