## Step 2 — `execute_read` Method

Add to `KbNavigationService` after `execute_glob`:

```python
_FULL_READ_MAX_LINES = 2000
_RANGE_READ_MAX_LINES = 500

def execute_read(
    self,
    user_id: str,
    document_id: str,
    start_line: int | None = None,
    end_line: int | None = None,
) -> KbReadResult:
    """Read full_markdown content of a document.

    start_line and end_line are 1-indexed and inclusive.
    If omitted: return full document (capped at 2000 lines).
    If provided: return the requested range (max 500 lines, validated upstream).
    """
    resp = (
        self.client.table("ose_raw_document_registry")
        .select("id,file_name,full_markdown")
        .eq("id", document_id)
        .eq("user_id", user_id)
        .limit(1)
        .execute()
    )
    rows = resp.data or []
    if not rows:
        raise KbNavigationError(
            f"Document {document_id!r} not found for this user."
        )
    row = rows[0]
    full_markdown: str | None = row.get("full_markdown")
    if full_markdown is None:
        raise KbNavigationError(
            "Document content not yet available. Ingestion may still be in progress."
        )

    lines = full_markdown.split("\n")
    total_lines = len(lines)

    if start_line is None:
        # Full read — cap at max lines
        actual_end = min(total_lines, self._FULL_READ_MAX_LINES)
        truncated = total_lines > self._FULL_READ_MAX_LINES
        content_lines = lines[:actual_end]
        return KbReadResult(
            document_id=document_id,
            name=row["file_name"],
            total_lines=total_lines,
            start_line=1,
            end_line=actual_end,
            truncated=truncated,
            content="\n".join(content_lines),
        )
    else:
        # Range read — start_line and end_line already validated upstream
        actual_end = min(end_line, total_lines)  # type: ignore[arg-type]
        content_lines = lines[start_line - 1 : actual_end]
        return KbReadResult(
            document_id=document_id,
            name=row["file_name"],
            total_lines=total_lines,
            start_line=start_line,
            end_line=actual_end,
            truncated=False,
            content="\n".join(content_lines),
        )
```

Note: `_FULL_READ_MAX_LINES` and `_RANGE_READ_MAX_LINES` are class-level constants on `KbNavigationService`. Add them just inside the class body, before `__init__`:

## Verification Steps

1. **Compile check:** `python -m compileall python-backend/services/folder_navigation.py` — no errors.

2. **Full read — short doc:** Create `KbReadResult` directly; simulate doc with 5 lines. Call `execute_read` via in-memory harness with `full_markdown="line1\nline2\nline3\nline4\nline5"`. Expect `total_lines=5`, `start_line=1`, `end_line=5`, `truncated=False`.

3. **Full read — truncation:** Simulate doc with 2500 lines. Expect `total_lines=2500`, `end_line=2000`, `truncated=True`, returned content has exactly 2000 lines.

4. **Range read — basic:** `start_line=3, end_line=5` on a 10-line doc. Expect `content` = lines 3–5, `total_lines=10`, `start_line=3`, `end_line=5`, `truncated=False`.

5. **Range read — end exceeds total:** `start_line=8, end_line=99` on a 10-line doc. Expect `end_line=10` (capped), `content` = lines 8–10.

6. **Not found:** Row lookup returns empty. Expect `KbNavigationError` with "not found for this user".

7. **Content not available:** Row found but `full_markdown=None`. Expect `KbNavigationError` with "not yet available".

8. **Serializer shape:** `read_result_to_dict(result)` returns dict with keys: `document_id`, `name`, `total_lines`, `start_line`, `end_line`, `truncated`, `content`.

9. **Single-line doc:** `full_markdown="only one line"`. `split("\n")` → 1 element. `total_lines=1`, `end_line=1`, `content="only one line"`.

10. **Empty content:** `full_markdown=""`. `split("\n")` → `[""]`, `total_lines=1`. Should return `content=""` and `end_line=1`.

## Notes

- `end_line` type hint is `int | None` in the method signature; the `type: ignore` comment on `min(end_line, total_lines)` is needed because mypy sees it as potentially None at that point. In practice, `start_line is None` check ensures `end_line` is not None when used.
- `_FULL_READ_MAX_LINES` and `_RANGE_READ_MAX_LINES` are class constants, not instance attributes. They are never accessed via `self._RANGE_READ_MAX_LINES` in the service method body because range validation happens in the endpoint (Pydantic model) — the service trusts that validated inputs arrive within bounds.
- The service does NOT enforce the 500-line range cap — that is the endpoint's responsibility via Pydantic validator. The service only enforces the 2000-line full-read cap.
