# 03-02-PLAN: Folder Tree Components

## Goal

Build the visual components for the filesystem-tree UI. These components are pure presentational/controlled — they receive state and callbacks as props, call nothing directly. All data fetching and mutation goes through the hook built in 03-01.

---

## Component Directory

Create all components in `components/pro-suite/os-engine/kb/`.

```
kb/
  FolderTree.tsx       — outer container; renders root header + all nodes
  FolderNode.tsx       — single folder row (expand/collapse, hover icons, right-click)
  FileNode.tsx         — single file row + inline metadata + delete confirm
  KbContextMenu.tsx    — portaled right-click context menu
  InlineNameInput.tsx  — shared inline input for create/rename
```

---

## Component 1: `InlineNameInput.tsx`

Shared by new-folder creation and rename. Renders an auto-focused text input inline.

```typescript
interface InlineNameInputProps {
  initialValue?: string;
  placeholder?: string;
  onConfirm: (name: string) => void;
  onCancel: () => void;
}
```

Behavior:
- Auto-focus on mount (`useEffect(() => ref.current?.focus(), [])`)
- `Enter` → calls `onConfirm(value.trim())` if value is non-empty; noop if empty
- `Escape` → calls `onCancel()`
- `onBlur` → calls `onCancel()` (clicking away cancels)

Styling: match AOS input tokens. Inline, not modal. Width matches available space in context (folder row width).

## Component 2: `KbContextMenu.tsx`

Right-click context menu. **Must be portaled** via `ReactDOM.createPortal(menu, document.body)` to avoid z-index and overflow clipping issues.

```typescript
export type KbContextMenuTarget =
  | { type: 'folder'; folderId: string; folderName: string }
  | { type: 'root' };

interface KbContextMenuProps {
  target: KbContextMenuTarget | null;       // null = closed
  position: { x: number; y: number };
  onCreateSubfolder: (parentId: string | null) => void;
  onRename: (folderId: string) => void;
  onDelete: (folderId: string) => void;
  onClose: () => void;
}
```

Behavior:
- When `target` is null, render nothing
- Render at `{ top: position.y, left: position.x }` via `position: fixed`
- Close on click-outside: `useEffect` adds a `mousedown` listener on `document`, fires `onClose`
- Close on `Escape` key
- Menu items by target type:
  - **folder**: "New subfolder", "Rename", "Delete" (with a divider before Delete)
  - **root**: "New folder" only

Styling:
- Background: `var(--bg-surface)` (cloud white)
- Border: `1px solid var(--aos-mist)`
- Border radius: `var(--radius-sm)`
- Box shadow: subtle (follow existing card shadow pattern — no glows)
- Min width: 160px
- Item padding: `8px 12px`
- Hover: `var(--bg-canvas)` background on item
- Delete item: `var(--aos-risk)` text color
- z-index: 9999

## Component 3: `FolderNode.tsx`

Single folder row in the tree. Manages its own hover state and right-click. Does NOT manage expansion state — that comes from props.

```typescript
interface FolderNodeProps {
  folder: FolderTreeNode;           // from 03-01 (includes .children)
  depth: number;                    // indent level (0 = root)
  isExpanded: boolean;
  isSelected: boolean;
  isRenaming: boolean;              // show InlineNameInput instead of name
  docs: RawDocument[];              // all docs for this user (filter by folder_id)
  expandedIds: Set<string>;
  selectedFolderId: string | null;

  onToggleExpand: (id: string) => void;
  onSelect: (id: string) => void;
  onContextMenu: (e: React.MouseEvent, target: KbContextMenuTarget) => void;
  onRenameConfirm: (id: string, name: string) => void;
  onRenameCancel: () => void;
  onOpenDoc: (doc: RawDocument) => void;
  onDeleteDoc: (docId: string) => Promise<void>;

  // Passed down for nested FileNodes
  pages: KnowledgePage[];
}
```

### Render structure:

```
<div style={{ paddingLeft: depth * 16 }}>
  {/* Folder row */}
  <div
    className="folder-row"
    onClick={() => { onToggleExpand(folder.id); onSelect(folder.id); }}
    onContextMenu={(e) => { e.preventDefault(); onContextMenu(e, { type: 'folder', folderId: folder.id, folderName: folder.name }); }}
  >
    <ChevronRight/ChevronDown size={14} />   {/* expand indicator */}
    <FolderIcon size={14} />                  {/* lucide Folder or FolderOpen */}
    {isRenaming
      ? <InlineNameInput initialValue={folder.name} onConfirm={(n) => onRenameConfirm(folder.id, n)} onCancel={onRenameCancel} />
      : <span>{folder.name}</span>
    }
    {/* Hover-reveal action icons — only show when not renaming */}
    {!isRenaming && <HoverActions folderId={folder.id} ... />}
  </div>

  {/* Expanded children */}
  {isExpanded && (
    <>
      {/* Child folders (recursive) */}
      {folder.children.map(child => (
        <FolderNode key={child.id} folder={child} depth={depth + 1} ... />
      ))}
      {/* Files in this folder */}
      {docsInFolder.map(doc => (
        <FileNode key={doc.id} doc={doc} depth={depth + 1} ... />
      ))}
      {/* Empty state */}
      {folder.children.length === 0 && docsInFolder.length === 0 && (
        <div style={{ paddingLeft: (depth + 1) * 16 }} className="empty-folder-hint">
          Empty folder
        </div>
      )}
    </>
  )}
</div>
```

Where `docsInFolder = docs.filter(d => d.folderId === folder.id)`.

### Hover actions (inline in FolderNode or a tiny sub-component):
- Pencil icon: triggers rename mode (`onRenameStart(folder.id)` — managed by parent `FolderTree`)
- Trash icon: triggers delete confirmation (a `window.confirm()` is acceptable for beta, or a simple `isConfirmingDelete` state on `FolderNode`)

### Folder row styling:
```css
/* Folder row */
display: flex; align-items: center; gap: 6px;
padding: 5px 8px;
border-radius: var(--radius-sm);
cursor: pointer;
color: var(--fg-1);
font-size: 0.875rem; /* text-sm */

/* Selected */
background: var(--bg-canvas);
color: var(--aos-brass);

/* Hover */
background: var(--bg-canvas);

/* Chevron + folder icon */
color: var(--fg-3);
flex-shrink: 0;
```

## Component 5: `FolderTree.tsx`

The outer container. Receives all state from the hook (via `UploadsView` props). Renders the root header and all top-level nodes.

```typescript
interface FolderTreeProps {
  tree: FolderTreeNode[];                    // from useKbFolderTree
  docs: RawDocument[];                       // all user docs
  pages: KnowledgePage[];
  selectedFolderId: string | null;
  expandedIds: Set<string>;

  onSelectFolder: (id: string | null) => void;
  onToggleExpand: (id: string) => void;
  onCreateFolder: (name: string, parentId?: string | null) => Promise<void>;
  onRenameFolder: (id: string, name: string) => Promise<void>;
  onDeleteFolder: (id: string) => Promise<void>;
  onOpenDoc: (doc: RawDocument) => void;
  onDeleteDoc: (docId: string) => Promise<void>;
}
```

Internal state on `FolderTree`:
- `renamingId: string | null` — which folder is in rename mode
- `creatingIn: string | null | 'root'` — `'root'` = new root folder, `string` = new subfolder under that ID
- `contextMenu: { target: KbContextMenuTarget; position: { x: number; y: number } } | null`

### Render structure:

```tsx
<div className="kb-folder-tree">
  {/* Root / "All Files" header row */}
  <div
    className={`folder-root-row ${selectedFolderId === null ? 'selected' : ''}`}
    onClick={() => onSelectFolder(null)}
    onContextMenu={(e) => { e.preventDefault(); openContextMenu(e, { type: 'root' }); }}
  >
    <Database size={14} />   {/* or Layers or FolderKanban — pick from lucide to match AOS feel */}
    <span>Knowledge Base</span>
    <span className="doc-count">{docs.length} files</span>
  </div>

  {/* Root-level files (folder_id === null) — only shown when root is selected or expanded */}
  {selectedFolderId === null && rootDocs.map(doc => (
    <FileNode key={doc.id} doc={doc} depth={1} ... />
  ))}

  {/* Folder tree */}
  {tree.map(folder => (
    <FolderNode
      key={folder.id}
      folder={folder}
      depth={1}
      isExpanded={expandedIds.has(folder.id)}
      isSelected={selectedFolderId === folder.id}
      isRenaming={renamingId === folder.id}
      docs={docs}
      expandedIds={expandedIds}
      selectedFolderId={selectedFolderId}
      onToggleExpand={onToggleExpand}
      onSelect={onSelectFolder}
      onContextMenu={handleContextMenu}
      onRenameConfirm={handleRenameConfirm}
      onRenameCancel={() => setRenamingId(null)}
      onOpenDoc={onOpenDoc}
      onDeleteDoc={onDeleteDoc}
      pages={pages}
    />
  ))}

  {/* Inline input for new root folder */}
  {creatingIn === 'root' && (
    <div style={{ paddingLeft: 16 }}>
      <InlineNameInput
        placeholder="Folder name"
        onConfirm={(name) => handleCreateConfirm(name, null)}
        onCancel={() => setCreatingIn(null)}
      />
    </div>
  )}

  {/* + New folder button */}
  <button
    onClick={() => setCreatingIn('root')}
    className="new-folder-btn"
  >
    <Plus size={13} />
    New folder
  </button>

  {/* Portaled context menu */}
  <KbContextMenu
    target={contextMenu?.target ?? null}
    position={contextMenu?.position ?? { x: 0, y: 0 }}
    onCreateSubfolder={handleCreateSubfolder}
    onRename={handleRenameStart}
    onDelete={handleDeleteFolder}
    onClose={() => setContextMenu(null)}
  />
</div>
```

Where:
- `rootDocs = docs.filter(d => d.folderId === null || d.folderId === undefined)`
- `handleContextMenu(e, target)` → sets `contextMenu` state
- `handleRenameStart(id)` → sets `renamingId = id`, closes context menu
- `handleRenameConfirm(id, name)` → calls `onRenameFolder(id, name)`, clears `renamingId`
- `handleCreateSubfolder(parentId)` → sets `creatingIn = parentId`, closes context menu
- `handleCreateConfirm(name, parentId)` → calls `onCreateFolder(name, parentId)`, clears `creatingIn`
- `handleDeleteFolder(id)` → calls `onDeleteFolder(id)`, closes context menu

## Verification Steps

1. **Build passes**: `npm run build` with no TypeScript errors on the new components.
2. **FolderTree renders empty state**: With no folders and no docs, renders "Knowledge Base" root row and "+ New folder" button.
3. **Create folder**: Click "+ New folder" → `InlineNameInput` appears at root → type name → Enter → folder appears in tree.
4. **Create subfolder**: Right-click on folder → "New subfolder" → `InlineNameInput` appears as first child → folder appears nested.
5. **Rename**: Hover over folder → pencil icon → `InlineNameInput` appears with current name → update → folder name changes in place.
6. **Delete**: Hover over folder → trash → confirmation → `onDeleteFolder` called → folder disappears.
7. **Expand/collapse**: Clicking a folder with children expands/collapses the children list.
8. **FileNode metadata expand**: Clicking the chevron on a file row opens the metadata panel below it (same content as current UploadsView expanded row).
9. **FileNode delete confirm**: Clicking trash on a file shows the "Type DELETE" confirmation inline.
10. **Context menu**: Right-click folder → menu appears at cursor position, portaled to body. Click outside closes it.
11. **No visual regressions**: AOS color tokens used throughout — no Tailwind default grays, no black backgrounds, no glows.
