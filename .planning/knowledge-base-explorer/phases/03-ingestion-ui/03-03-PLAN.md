# 03-03-PLAN: UploadsView Integration

## Goal

Wire the `FolderTree` component into `UploadsView.tsx` and update its parent component to drive folder state from `useKbFolderTree`. The flat doc table is replaced by `FolderTree`. The upload zone gets folder-targeting behavior.

---

## Step 1 — Locate the Parent Component

Before touching `UploadsView`, find where `<UploadsView` is rendered:

```bash
grep -r "UploadsView" src/ --include="*.tsx" -l
```

Read that parent component in full. It will be the place where:
- `docs` state is managed (fetched from Supabase)
- `onUpload` handler is defined (calls `uploadRawDocument`)
- `onDeleteDoc` handler is defined
- `pages` state is provided

This parent is where `useKbFolderTree` gets imported and used.

---

## Step 3 — Update Upload Zone in `UploadsView`

**3a. Add folder badge above the "Browse files" button:**

When `selectedFolderId` is not null, show a small label in the upload zone:

```tsx
{selectedFolderId && (
  <p className="text-xs text-[var(--fg-2)] mb-1">
    Uploading to: <span className="font-medium text-[var(--fg-1)]">{folderName}</span>
    <button onClick={() => onSelectFolder(null)} className="ml-2 text-[var(--fg-4)] hover:text-[var(--fg-2)]">✕</button>
  </p>
)}
```

Where `folderName` is derived by finding the folder in `folderTree` (walk the tree or keep a flat lookup). The ✕ button clears the folder selection.

**3b. Pass `selectedFolderId` through to `onUpload`:**

```tsx
// uploadFile function — update both call sites:
const uploadFile = async (file?: File) => {
  if (!file || busy) return;
  setBusy(true);
  setError(null);
  try {
    await onUpload(file, selectedFolderId);   // ← pass folderId
  } catch (err) {
    setError(err instanceof Error ? err.message : 'Upload failed.');
  } finally {
    setBusy(false);
    if (fileInputRef.current) fileInputRef.current.value = '';
  }
};
```

The drag-and-drop handler already calls `uploadFile(event.dataTransfer.files[0])` — it inherits `selectedFolderId` from closure. No change needed there.

## Step 4 — Replace Flat Table with FolderTree

In `UploadsView.tsx`, remove the `{docs.length === 0 ? ... : <Table>...</Table>}` block entirely. Replace with:

```tsx
{foldersLoading ? (
  <div className="text-sm text-[var(--fg-3)] py-4">Loading folders...</div>
) : (
  <FolderTree
    tree={folderTree}
    docs={docs}
    pages={pages}
    selectedFolderId={selectedFolderId}
    expandedIds={expandedIds}
    onSelectFolder={onSelectFolder}
    onToggleExpand={onToggleExpand}
    onCreateFolder={onCreateFolder}
    onRenameFolder={onRenameFolder}
    onDeleteFolder={onDeleteFolder}
    onOpenDoc={onOpenDoc}
    onDeleteDoc={onDeleteDoc}
  />
)}
```

The `FolderTree` handles its own empty state ("No files uploaded yet" equivalent) — when there are no folders and no docs, it shows a minimal empty state below the "Knowledge Base" root row.

## Verification Steps

1. **Build passes**: `npm run build` — no TypeScript errors.
2. **Uploads page renders**: Navigate to OS Engine → Uploads. Page loads without errors. Upload zone is visible. "Knowledge Base" root row appears below upload zone.
3. **No docs / no folders**: Shows "Knowledge Base" row + "+ New folder" button + no file rows.
4. **Create folder**: Click "+ New folder", type a name, Enter. Folder appears in tree.
5. **Upload to root**: Upload a file while no folder is selected. File appears under "Knowledge Base" in tree after ingestion status updates.
6. **Upload to folder**: Click a folder to select it. Upload zone shows "Uploading to: [folder name]". Upload a file. After refresh/reload, file appears under that folder.
7. **Expand folder**: Click a folder that contains files. Files appear indented below the folder row.
8. **Metadata expand**: Click the expand chevron on a file row. Metadata panel opens below (same content as before — summary, type, area, period, metrics, etc.).
9. **Delete file**: Click trash on a file row, type DELETE, confirm. File disappears.
10. **Rename folder**: Hover over folder, click pencil. Inline input appears with current name. Change name, Enter. Name updates.
11. **Delete folder**: Hover over folder, click trash. Folder and its tree entries disappear. Files that were in the folder move to root (visible under "Knowledge Base").
12. **Right-click context menu**: Right-click folder → menu appears at cursor, portaled to body (inspect DOM to confirm it's a direct child of `<body>`). Click outside closes it.
13. **Existing behaviors preserved**: `onOpenDoc` opens the doc panel. StatusPill renders. Connected pages display. All unchanged behaviors work.
14. **No visual regressions**: AOS tokens used, no forbidden patterns (no `gray-*`, no black, no glows).
