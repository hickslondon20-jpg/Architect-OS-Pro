## Step 3 — New Endpoint Functions

Add after the `kb_tree` function (currently around line 383) and before `@app.get("/api/agent-capabilities"...)`:

```python
@app.post("/api/tools/kb-grep", response_model=KbGrepResponse, dependencies=[Depends(require_ingest_secret)])
def kb_grep(payload: KbGrepRequest) -> KbGrepResponse:
    try:
        result = KbNavigationService.from_env().execute_grep(
            user_id=payload.user_id,
            pattern=payload.pattern,
            folder_id=payload.folder_id,
            limit=payload.limit,
        )
    except KbNavigationError as exc:
        raise HTTPException(status_code=status.HTTP_400_BAD_REQUEST, detail=str(exc)) from exc
    except VectorStoreError as exc:
        raise HTTPException(status_code=status.HTTP_502_BAD_GATEWAY, detail=str(exc)) from exc
    return KbGrepResponse(**grep_result_to_dict(result))


@app.post("/api/tools/kb-glob", response_model=KbGlobResponse, dependencies=[Depends(require_ingest_secret)])
def kb_glob(payload: KbGlobRequest) -> KbGlobResponse:
    try:
        result = KbNavigationService.from_env().execute_glob(
            user_id=payload.user_id,
            pattern=payload.pattern,
            folder_id=payload.folder_id,
            limit=payload.limit,
        )
    except KbNavigationError as exc:
        raise HTTPException(status_code=status.HTTP_400_BAD_REQUEST, detail=str(exc)) from exc
    except VectorStoreError as exc:
        raise HTTPException(status_code=status.HTTP_502_BAD_GATEWAY, detail=str(exc)) from exc
    return KbGlobResponse(**glob_result_to_dict(result))
```

## Error Handling Notes

- `KbNavigationError` on grep = invalid regex → `400 Bad Request` (not 404). This differs from ls/tree where KbNavigationError means "folder not found" → 404. Grep raises KbNavigationError for invalid patterns, not missing resources.
- `KbNavigationError` on glob = this won't normally raise (glob never validates the pattern against a DB constraint), but keep the 400 handler for completeness and future-proofing.
- `VectorStoreError` → `502 Bad Gateway` (same as all other tool endpoints).

## Key Difference from Phase 4 Endpoints

Phase 4 `KbNavigationError` → 404 (folder not found).  
Phase 5 grep `KbNavigationError` → 400 (invalid pattern — client error, not a resource-not-found).

This is intentional and correct. Do not accidentally use 404 for grep/glob errors.
