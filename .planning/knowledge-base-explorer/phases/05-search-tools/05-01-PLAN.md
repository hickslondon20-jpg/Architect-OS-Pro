## Step 3 ‚Äî Helper: `_collect_folder_subtree_ids`

Add as a module-level function (NOT a method on the class ‚Äî it is a pure data transformation):

```python
def _collect_folder_subtree_ids(
    all_folder_rows: list[dict[str, Any]],
    root_folder_id: str,
) -> set[str]:
    """Return all folder IDs in the subtree rooted at root_folder_id (inclusive)."""
    children_by_parent: dict[str, list[str]] = {}
    for row in all_folder_rows:
        parent = row.get("parent_id")
        if parent:
            children_by_parent.setdefault(parent, []).append(row["id"])

    result: set[str] = set()
    queue = [root_folder_id]
    while queue:
        current = queue.pop()
        result.add(current)
        queue.extend(children_by_parent.get(current, []))
    return result
```

## Step 4 ‚Äî `execute_grep` Method

Add to `KbNavigationService` after `execute_tree`:

```python
def execute_grep(
    self,
    user_id: str,
    pattern: str,
    folder_id: str | None = None,
    limit: int = 50,
) -> KbGrepResult:
    """Search full_markdown of user documents for regex pattern.

    folder_id=None searches all user docs.
    folder_id=uuid searches that folder's entire subtree (recursive).
    Returns matching document identifiers ‚Äî not content excerpts.
    """
    limit = max(1, min(limit, 100))

    # Validate pattern before sending to Postgres
    try:
        re.compile(pattern)
    except re.error as exc:
        raise KbNavigationError(f"Invalid regex pattern: {exc}") from exc

    # Build base query
    query = (
        self.client.table("ose_raw_document_registry")
        .select("id,file_name,file_type,status,folder_id")
        .eq("user_id", user_id)
        .neq("status", "deleted")
        .filter("full_markdown", "~*", pattern)
        .limit(limit)
    )

    if folder_id is not None:
        # Collect subtree folder IDs
        all_folders_resp = (
            self.client.table("kb_folders")
            .select("id,parent_id")
            .eq("user_id", user_id)
            .execute()
        )
        subtree_ids = _collect_folder_subtree_ids(
            all_folders_resp.data or [], folder_id
        )
        query = query.in_("folder_id", list(subtree_ids))

    resp = query.execute()
    matches = [
        KbSearchMatch(
            id=row["id"],
            name=row["file_name"],
            file_type=row.get("file_type") or "",
            status=row.get("status") or "",
            folder_id=row.get("folder_id"),
        )
        for row in resp.data or []
    ]
    return KbGrepResult(
        pattern=pattern,
        scope_folder_id=folder_id,
        match_count=len(matches),
        matches=matches,
    )
```

## Step 5 ‚Äî `execute_glob` Method

Add to `KbNavigationService` after `execute_grep`:

```python
def execute_glob(
    self,
    user_id: str,
    pattern: str,
    folder_id: str | None = None,
    limit: int = 200,
) -> KbGlobResult:
    """Match document filenames against a glob pattern.

    folder_id=None searches all user docs.
    folder_id=uuid searches that folder's entire subtree (recursive).
    Uses Python fnmatch ‚Äî supports *, ?, [seq]. Case-insensitive.
    """
    limit = max(1, min(limit, 200))

    # Fetch candidate docs
    query = (
        self.client.table("ose_raw_document_registry")
        .select("id,file_name,file_type,status,folder_id")
        .eq("user_id", user_id)
        .neq("status", "deleted")
    )

    if folder_id is not None:
        all_folders_resp = (
            self.client.table("kb_folders")
            .select("id,parent_id")
            .eq("user_id", user_id)
            .execute()
        )
        subtree_ids = _collect_folder_subtree_ids(
            all_folders_resp.data or [], folder_id
        )
        query = query.in_("folder_id", list(subtree_ids))

    resp = query.execute()

    pattern_lower = pattern.lower()
    matches: list[KbSearchMatch] = []
    for row in resp.data or []:
        if fnmatch.fnmatchcase(row["file_name"].lower(), pattern_lower):
            matches.append(
                KbSearchMatch(
                    id=row["id"],
                    name=row["file_name"],
                    file_type=row.get("file_type") or "",
                    status=row.get("status") or "",
                    folder_id=row.get("folder_id"),
                )
            )
            if len(matches) >= limit:
                break

    return KbGlobResult(
        pattern=pattern,
        scope_folder_id=folder_id,
        match_count=len(matches),
        matches=matches,
    )
```

## Verification Steps

1. **Compile check**: `python -m compileall python-backend/services/folder_navigation.py` ‚Äî no errors.
2. **`_collect_folder_subtree_ids` ‚Äî flat**: Given `[{id: "A", parent_id: null}, {id: "B", parent_id: "A"}]`, call with `"A"` ‚Äî returns `{"A", "B"}`.
3. **`_collect_folder_subtree_ids` ‚Äî deep**: 3-level chain A‚ÜíB‚ÜíC, call with `"A"` ‚Äî returns `{"A", "B", "C"}`.
4. **`_collect_folder_subtree_ids` ‚Äî leaf**: Call with `"C"` (no children) ‚Äî returns `{"C"}`.
5. **`execute_grep` ‚Äî valid pattern, no matches**: Pattern that won't match any doc ‚Äî returns `match_count=0`, `matches=[]`.
6. **`execute_grep` ‚Äî invalid regex**: Pattern `"[unclosed"` ‚Äî raises `KbNavigationError` with "Invalid regex pattern" message.
7. **`execute_grep` ‚Äî null full_markdown docs not included**: Service-code harness confirming that docs without `full_markdown` (simulated as if Postgres filter returns only rows where `~*` is true) are excluded.
8. **`execute_glob` ‚Äî `*.pdf`**: Given docs `["report.pdf", "data.csv", "plan.pdf"]`, pattern `"*.pdf"` returns the two PDF files.
9. **`execute_glob` ‚Äî `report*`**: Given above docs, returns `["report.pdf"]` only.
10. **`execute_glob` ‚Äî `*`**: Returns all docs.
11. **`execute_glob` ‚Äî case insensitive**: Pattern `"*.PDF"` matches `"report.pdf"`.
12. **`execute_glob` ‚Äî limit enforced**: Given 10 docs matching `"*"`, limit=3 ‚Üí returns 3 matches.

## Notes

- `_collect_folder_subtree_ids` is a pure function with no Supabase dependency ‚Äî easy to unit test without any mocking.
- The `execute_grep` `folder_id` path makes 2 Supabase calls (1 for folders, 1 for docs with regex filter). The unscoped path makes 1 call. This is acceptable.
- PostgreSQL `~*` regex and Python `re.compile` regex are not identical ‚Äî lookbehind assertions and some advanced features differ. For agent use (simple patterns like `revenue`, `Q[34]`, `client.*report`) this is not a concern.
- `execute_glob` fetches ALL non-deleted docs for the user (or subtree) and filters in Python. For beta scale (tens to hundreds of docs), this is fine. At thousands of docs, push the limit filter down to Postgres using a `LIKE` approximation ‚Äî but that's a v2 optimization.
