## Changes

### `lib/osEngineApi.ts`

**1. Update `RawDocument` type (or wherever document types are defined)**

Add `folder_id` to the document type used in this file. If types are in a separate file, add it there and import. If inline, add it directly.

```typescript
folder_id?: string | null;
```

**2. Update `uploadRawDocument` function signature**

Current signature (approximate):
```typescript
export const uploadRawDocument = async (file: File, ...) => { ... }
```

New signature — add optional `folderId` parameter:
```typescript
export const uploadRawDocument = async (
  file: File,
  folderId?: string | null,
  // ... other existing params unchanged
) => { ... }
```

**3. Pass `folder_id` in the `ose_raw_document_registry` insert**

Locate the Supabase insert for the registry row (currently around line 274–280 in the file). Add `folder_id`:

```typescript
const insertResult = await supabase
  .from('ose_raw_document_registry')
  .insert({
    id: docId,
    user_id: userId,
    file_name: file.name,
    file_type: fileType,
    // ... existing fields ...
    folder_id: folderId ?? null,   // ADD THIS
  });
```

**4. Also update the duplicate-row insert** (around line 244–250)

The duplicate path also inserts a registry row. Add `folder_id` there too:
```typescript
const duplicateInsert = await supabase
  .from('ose_raw_document_registry')
  .insert({
    // ... existing fields ...
    folder_id: folderId ?? null,   // ADD THIS
  });
```

**5. The `/api/ingest` call does NOT change**

The ingest call (around line 309–320) sends `document_id`, `user_id`, `storage_path`, `file_name`, `file_type` to the backend. `folder_id` is already persisted in the registry row at insert time — the backend does not need it.

**6. Update callers**

All existing call sites of `uploadRawDocument` pass `folderId` as `undefined` or `null` by default (parameter is optional). No caller is broken. The Phase 3 UI will be the first to pass an actual `folderId` value.
