# Plan 02-03 — Ingestion Service: Store full_markdown During Processing

**Phase:** 2 — Document-Folder Integration
**Plan:** 3 of 4
**Requirements:** DOC-03 (store full extracted markdown alongside chunks)
**Depends on:** 02-01 complete (`full_markdown` column must exist on `ose_raw_document_registry`)

## Goal

After Docling extracts text from a document, store the full extracted text as `full_markdown` in `ose_raw_document_registry`. This enables the grep/read tools in Phases 5–6 to operate without reconstructing text from chunks.

---

2. Read `python-backend/main.py` `_process_ingestion` function — understand where `processed.text` is available and which `store.*` calls surround it.
3. Confirm `full_markdown` column exists on `ose_raw_document_registry` (02-01 done).

## Changes

### `python-backend/services/vector_store.py`

Add one new method to the `VectorStore` class. Place it after `mark_parser_complete`:

```python
def store_full_markdown(self, document_id: str, user_id: str, text: str) -> None:
    """Persist the full extracted markdown text for grep/read tool access."""
    self._update_document(
        document_id,
        user_id,
        {"full_markdown": text or ""},
    )
```

No other changes to `vector_store.py`.

---

### `python-backend/main.py`

In `_process_ingestion`, add a `store_full_markdown` call immediately after `mark_parser_complete`. The call must be inside the existing try/except block so failures are handled consistently.

Current flow (relevant section):
```python
processed = process_document_bytes(...)
store.mark_parser_complete(payload.document_id, payload.user_id, processed.metadata)
# ... metadata extraction ...
store.replace_document_chunks(...)
store.mark_ingested(...)
```

New flow — add one line after `mark_parser_complete`:
```python
processed = process_document_bytes(...)
store.mark_parser_complete(payload.document_id, payload.user_id, processed.metadata)
store.store_full_markdown(payload.document_id, payload.user_id, processed.text)  # ADD
# ... metadata extraction ...
store.replace_document_chunks(...)
store.mark_ingested(...)
```

**Important:** `processed.text` is the `ProcessedDocument.text` field — the full concatenated extracted text from `doc_processor.py`. It is always a `str`. For CSV files or plain text the parser returns the raw text. For Docling-processed files it is the markdown export. Do not modify `doc_processor.py`.
