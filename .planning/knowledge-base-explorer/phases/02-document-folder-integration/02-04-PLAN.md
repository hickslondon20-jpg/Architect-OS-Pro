## New File: `python-backend/routers/kb_documents.py`

### Document Move Endpoint

```
PATCH /kb/documents/{document_id}/folder
Request body: { "folder_id": UUID | null }
Auth: Bearer JWT (Supabase)
Response 200: { "document_id": str, "folder_id": str | null }
Response 404: document not found or not owned by user
Response 400: folder_id exists but not owned by user
Response 401: missing/invalid token
```

**Logic:**
1. Extract `user_id` from JWT (reuse `get_current_user_id` from `kb_folders`)
2. Verify document exists and belongs to `user_id`:
   - SELECT from `ose_raw_document_registry` where `id = document_id AND user_id = user_id`
   - If not found → 404
3. If `folder_id` is provided (not null), verify it exists and belongs to `user_id`:
   - SELECT from `kb_folders` where `id = folder_id AND user_id = user_id`
   - If not found → 400
4. UPDATE `ose_raw_document_registry` SET `folder_id = folder_id` where `id = document_id AND user_id = user_id`
5. Return `{ "document_id": str(document_id), "folder_id": str(folder_id) if folder_id else None }`

**Pydantic models:**
```python
class DocumentMovePayload(BaseModel):
    folder_id: UUID | None = None

class DocumentMoveResponse(BaseModel):
    document_id: UUID
    folder_id: UUID | None
```

## Modify: `python-backend/routers/kb_folders.py`

### Folder Move Endpoint

Add to existing `kb_folders.py`:

```
PATCH /kb/folders/{folder_id}/parent
Request body: { "parent_id": UUID | null }
Auth: Bearer JWT (Supabase)
Response 200: FolderResponse (updated folder row)
Response 404: folder not found or not owned by user
Response 400: parent_id not found, not owned by user, or creates a cycle
Response 401: missing/invalid token
```

**Logic:**
1. Extract `user_id` from JWT
2. Verify the folder being moved exists and belongs to `user_id`
   - Use `_find_owned_folder(client, folder_id, user_id)` — if None → 404
3. If `parent_id` is provided (not null):
   a. Verify `parent_id` exists and belongs to `user_id` → if not → 400
   b. Run cycle check: ensure `parent_id` is NOT a descendant of `folder_id`
4. UPDATE `kb_folders` SET `parent_id = parent_id` where `id = folder_id AND user_id = user_id`
5. Return `FolderResponse`

**Cycle prevention — Python implementation:**

```python
def _would_create_cycle(
    client: Client,
    folder_id: UUID,
    new_parent_id: UUID,
    user_id: UUID,
) -> bool:
    """
    Returns True if setting folder_id.parent_id = new_parent_id would create a cycle.
    A cycle exists if new_parent_id is folder_id itself, or is a descendant of folder_id.
    Walk up the ancestor chain of new_parent_id — if we ever hit folder_id, it's a cycle.
    """
    if new_parent_id == folder_id:
        return True

    visited: set[str] = set()
    current_id: str | None = str(new_parent_id)

    while current_id is not None:
        if current_id in visited:
            break  # Shouldn't happen if DB is consistent, but guard infinite loop
        visited.add(current_id)

        response = (
            client.table("kb_folders")
            .select("parent_id")
            .eq("id", current_id)
            .eq("user_id", str(user_id))
            .limit(1)
            .execute()
        )
        rows = response.data or []
        if not rows:
            break  # Reached a folder we don't own or that doesn't exist

        parent = rows[0].get("parent_id")
        if parent is None:
            break  # Reached root

        if parent == str(folder_id):
            return True  # Cycle detected

        current_id = parent

    return False
```

   ```

   { "folder_id": "<folder_uuid>" }

   ```
   Verify in DB: `SELECT folder_id FROM ose_raw_document_registry WHERE id = '<doc_id>';`

2. Move document to root (folder_id = null):
   ```
   PATCH /kb/documents/{doc_id}/folder
   { "folder_id": null }
   → 200, folder_id is null
   ```

3. Document not found → 404.

4. Folder not owned by user → 400.

**Folder move:**

5. Create parent folder A and child folder B (B inside A). Create folder C at root. Move C into A:
   ```
   PATCH /kb/folders/{C_id}/parent
   { "parent_id": "<A_id>" }
   → 200, parent_id updated
   ```

6. Move folder to root (parent_id = null):
   ```
   PATCH /kb/folders/{C_id}/parent
   { "parent_id": null }
   → 200, parent_id is null
   ```

7. **Cycle prevention — direct:**
   ```
   PATCH /kb/folders/{A_id}/parent
   { "parent_id": "<A_id>" }   ← moving folder into itself
   → 400
   ```

8. **Cycle prevention — indirect:**
   ```
   Folder A contains B, B contains C.
   PATCH /kb/folders/{A_id}/parent
   { "parent_id": "<C_id>" }   ← would make A a child of its own descendant
   → 400
   ```
