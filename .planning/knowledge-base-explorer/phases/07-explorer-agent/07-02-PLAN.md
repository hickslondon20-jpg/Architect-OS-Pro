## Step 3 — Add `_handle_kb_explorer` Method

Add after `_handle_structured_data`:

```python
def _handle_kb_explorer(self, context: AgentContextBundle) -> dict[str, Any]:
    """Run the KB Explorer sub-agent tool-use loop."""
    exploration = KbExplorerService(self.store).run_exploration(
        user_id=context.user_id,
        task_summary=context.task_summary,
        max_rounds=5,
    )

    findings: list[dict[str, Any]] = []
    for step in exploration.tool_steps:
        findings.append(
            {
                "type": "tool_step",
                "title": f"Tool: {step['tool_name']}",
                "summary": step.get("summary", ""),
                "source_id": None,
                "error": step.get("error"),
            }
        )
    findings.append(
        {
            "type": "explorer_synthesis",
            "title": "Explorer synthesis",
            "summary": exploration.summary,
            "source_id": None,
        }
    )

    # Build citations from documents the agent actually read
    sources = [
        AgentSourceRef(
            source_kind="raw_document",
            source_id=doc_id,
            source_label=exploration.referenced_doc_names.get(doc_id, doc_id),
            source_metadata={"referenced_via": "kb_read"},
            citation_payload={
                "document_id": doc_id,
                "label": exploration.referenced_doc_names.get(doc_id, doc_id),
            },
        )
        for doc_id in exploration.referenced_doc_ids
    ]

    result_summary = exploration.summary[:500] if exploration.summary else "KB exploration complete."
    confidence = 0.75 if exploration.truncated else 0.85
    return _handler_result(result_summary, findings, sources, confidence=confidence)
```

## Notes

- `_handler_result()` is already defined in `sub_agent_orchestrator.py` as a module-level function. No changes needed to it.
- `KbExplorerService(self.store)` — passes the existing `VectorStore` instance, same pattern as `StructuredQueryService(self.store)` in `_handle_structured_data`.
- The `start_run()` method already records a step 1 (context_build) and step 2 (result) around every handler call. `_handle_kb_explorer` does NOT need to record its own steps to `agent_delegation_steps` — the tool steps are captured in `exploration.tool_steps` and appear in `findings`, which are stored via `structured_result["findings"]` in step 2. If per-tool DB step recording is needed in the future, it can be added to `_handle_kb_explorer` via additional `_create_step()` calls.
