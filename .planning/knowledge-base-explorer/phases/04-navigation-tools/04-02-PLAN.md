# 04-02-PLAN: FastAPI Endpoints for kb-ls and kb-tree

## Goal

Expose `KbNavigationService.execute_ls` and `execute_tree` as FastAPI endpoints in `main.py`, following the identical pattern to `POST /api/tools/structured-query` and `POST /api/tools/web-search`. Add `KbNavigationError` handling.

---

## Step 1 — Import in `main.py`

Add to the imports at the top of `main.py`:

```python
from services.folder_navigation import KbNavigationError, KbNavigationService
```

---

## Step 2 — Pydantic Request/Response Models

Add to `main.py` alongside the other request/response model classes:

```python
class KbLsRequest(BaseModel):
    user_id: str = Field(..., min_length=1)
    folder_id: str | None = Field(default=None)   # None = root


class KbLsResponse(BaseModel):
    folder_id: str | None
    folder_name: str
    item_count: int
    items: list[dict]   # [{type, id, name} or {type, id, name, file_type, status}]


class KbTreeRequest(BaseModel):
    user_id: str = Field(..., min_length=1)
    folder_id: str | None = Field(default=None)   # None = root
    depth: int = Field(default=3, ge=1, le=10)
    limit: int = Field(default=200, ge=1, le=500)


class KbTreeResponse(BaseModel):
    folder_id: str | None
    folder_name: str
    depth_requested: int
    total_items: int
    truncated: bool
    tree: list[dict]   # nested [{type, id, name, children?} ...]
```

---

## Step 3 — Endpoints in `main.py`

Add after the existing `/api/tools/web-search` endpoint:

```python
@app.post(
    "/api/tools/kb-ls",
    response_model=KbLsResponse,
    dependencies=[Depends(require_ingest_secret)],
)
def kb_ls(payload: KbLsRequest) -> KbLsResponse:
    try:
        result = KbNavigationService.from_env().execute_ls(
            user_id=payload.user_id,
            folder_id=payload.folder_id,
        )
    except KbNavigationError as exc:
        raise HTTPException(status_code=status.HTTP_404_NOT_FOUND, detail=str(exc)) from exc
    except VectorStoreError as exc:
        raise HTTPException(status_code=status.HTTP_502_BAD_GATEWAY, detail=str(exc)) from exc
    from services.folder_navigation import ls_result_to_dict
    data = ls_result_to_dict(result)
    return KbLsResponse(**data)


@app.post(
    "/api/tools/kb-tree",
    response_model=KbTreeResponse,
    dependencies=[Depends(require_ingest_secret)],
)
def kb_tree(payload: KbTreeRequest) -> KbTreeResponse:
    try:
        result = KbNavigationService.from_env().execute_tree(
            user_id=payload.user_id,
            folder_id=payload.folder_id,
            depth=payload.depth,
            limit=payload.limit,
        )
    except KbNavigationError as exc:
        raise HTTPException(status_code=status.HTTP_404_NOT_FOUND, detail=str(exc)) from exc
    except VectorStoreError as exc:
        raise HTTPException(status_code=status.HTTP_502_BAD_GATEWAY, detail=str(exc)) from exc
    from services.folder_navigation import tree_result_to_dict
    data = tree_result_to_dict(result)
    return KbTreeResponse(**data)
```

## Verification Steps

1. **Compile check**: `python -m compileall python-backend` — no errors.
2. **Server starts**: `uvicorn main:app --reload` starts without import errors.
3. **Health check**: `GET /api/health` → `{"ok": true}` (confirms existing endpoints unaffected).
4. **`/api/tools/kb-ls` — root, no auth**: `POST /api/tools/kb-ls` without `X-Ingest-Secret` → `401`.
5. **`/api/tools/kb-ls` — root, valid auth**: `POST /api/tools/kb-ls` with valid secret and `{"user_id": "<test_id>", "folder_id": null}` → `200` with `KbLsResponse` shape.
6. **`/api/tools/kb-ls` — specific folder**: POST with a real `folder_id` → returns immediate children only.
7. **`/api/tools/kb-ls` — nonexistent folder**: POST with a UUID that doesn't exist for this user → `200` with empty items (not a 404, since `execute_ls` does not validate folder existence — it just returns empty).
8. **`/api/tools/kb-tree` — defaults**: POST with `{"user_id": "...", "folder_id": null}` → returns tree with `depth_requested: 3`.
9. **`/api/tools/kb-tree` — depth param**: POST with `{"depth": 1}` → tree shows only root-level items, no children nested.
10. **`/api/tools/kb-tree` — limit truncation**: POST with `{"limit": 2}` on a tree with 5+ items → `truncated: true`, `total_items: 2`.
11. **OpenAPI docs**: Navigate to `/docs` — both new endpoints appear in the `default` group with correct request/response schema.

## Notes

- The `from services.folder_navigation import ls_result_to_dict` imports inside the handler are fine for clarity. If linters flag it, move them to the top-level import block — no functional difference.
- `KbLsResponse.items` and `KbTreeResponse.tree` use `list[dict]` instead of typed models to avoid a deeply nested Pydantic model definition. The response shape is stable and documented in CONTEXT.md.
- If the `folder_id` field on `ose_raw_document_registry` is not yet populated for existing test documents (uploaded before Phase 2), those documents will appear at root level (folder_id IS NULL) — this is correct behavior.
